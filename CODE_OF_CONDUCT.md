# Code of Conduct

ORIN follows a contributor code of conduct adapted from the Contributor Covenant 2.1.

## Our Pledge

We want ORIN to be a respectful, practical, and welcoming engineering community. Contributors, maintainers, and users are expected to collaborate without harassment, intimidation, personal attacks, or exclusionary behavior.

## Expected Behavior

Examples of constructive participation include:

- Treating other people with respect, even during technical disagreement
- Giving and accepting feedback with specific reasoning
- Staying focused on the project goal and the evidence in code, tests, docs, or reproducible reports
- Taking responsibility for mistakes and correcting them transparently
- Protecting private information and credentials

## Unacceptable Behavior

Unacceptable behavior includes:

- Harassment, threats, insults, or sustained disruption
- Sexualized language or attention in project spaces
- Publishing private information without explicit permission
- Personal, political, or identity-based attacks
- Sharing secrets, credentials, private logs, or user data in issues, PRs, discussions, or commits

## Scope

This code applies in ORIN project spaces, including issues, pull requests, discussions, code review, documentation, and community channels. It also applies when someone officially represents the project in public.

## Reporting

Report conduct concerns by opening a GitHub issue with sensitive details removed, or by contacting the repository owner through their GitHub profile if private handling is needed. Do not post secrets, credentials, private logs, or personal data in public reports.

Maintainers will review reports fairly, preserve reporter privacy where possible, and take action proportional to the situation. Actions may include clarification, warning, content removal, temporary restriction, or permanent removal from project spaces.
