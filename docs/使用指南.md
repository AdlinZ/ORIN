# ORIN 使用指南

本文档面向当前仓库实现，重点说明“从哪里进入功能”和“联调时先看什么”。涉及详细字段时，请以 OpenAPI 和代码为准。

## 快速访问

- 前端开发地址：`http://localhost:5173`
- 后端地址：`http://localhost:8080`
- AI 引擎默认地址：`http://localhost:8000`
- OpenAPI / Swagger UI：`http://localhost:8080/swagger-ui/index.html`

## 推荐启动方式

### 一键启动

```bash
cd /path/to/ORIN
./manage.sh start
```

说明：

- `manage.sh` 会尝试启动 MySQL、后端、前端、Python AI 引擎
- 默认端口是前端 `5173`、后端 `8080`、AI 引擎 `8000`
- 脚本默认按本地开发场景编写，更完整的环境配置见 [部署指南.md](/Users/adlin/Documents/Code/ORIN/docs/部署指南.md)

### 分别启动

```bash
cd orin-backend
mvn spring-boot:run
```

```bash
cd orin-ai-engine
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

```bash
cd orin-frontend
npm install
npm run dev
```

## 当前前端导航

当前主导航按模块分为四大块：

1. 智能体
2. 知识中心
3. 监控运维
4. 系统管理

如果你在旧文档里看到 `/dashboard/agent/*`、`/dashboard/workflow/*` 之类路径，不用惊讶。前端路由里保留了重定向兼容，但新的入口以 `orin-frontend/src/router/routes.js` 为准。

## 常用功能入口

### 1. 智能体

常用页面：

- 智能体列表：`/dashboard/agents/list`
- 智能体接入：`/dashboard/agents/onboard`
- 智能体控制台：`/dashboard/agents/console/:id`
- 会话记录：`/dashboard/agents/chat-logs`
- 协作任务：`/dashboard/agents/collaboration`
- 工作流入口：`/dashboard/agents/workflows`

适用场景：

- 接入外部 Agent 或模型服务
- 查看聊天记录和运行状态
- 调试协作任务或工作流执行

### 2. 知识中心

常用页面：

- 知识库列表：`/dashboard/knowledge/list`
- 新建知识库：`/dashboard/knowledge/create`
- 知识库详情：`/dashboard/knowledge/detail/:id`
- 检索实验室：`/dashboard/knowledge/retrieval-lab`
- 向量实验室：`/dashboard/knowledge/embedding-lab`
- 视觉实验室：`/dashboard/knowledge/vlm-lab`
- 知识图谱：`/dashboard/knowledge/graph`

现状说明：

- 文档上传、解析、分块、向量化相关能力在这部分收口
- 知识图谱页面已具备独立视图，但不要把它默认为“完整交付能力”，联调前建议先看当前实现和后端实体/仓储

### 3. 监控运维

常用页面：

- 监控总览：`/dashboard/monitor/dashboard`
- Token 统计：`/dashboard/monitor/tokens`
- 成本统计：`/dashboard/monitor/costs`
- 时延分析：`/dashboard/monitor/latency`
- 错误统计：`/dashboard/monitor/errors`
- Trace 查看：`/dashboard/monitor/traces`

适用场景：

- 观察调用链、成本、错误和任务堆积

### 4. 系统管理

常用页面：

- 用户管理：`/dashboard/system/users`
- 角色管理：`/dashboard/system/roles`

## 常见联调方式

### 看接口是否存在

优先打开 Swagger：

```bash
open http://localhost:8080/swagger-ui/index.html
```

如果只是快速核对控制器入口，可以直接搜后端注解：

```bash
rg "@RequestMapping|@GetMapping|@PostMapping" orin-backend/src/main/java/com/adlin/orin
```

### 看页面入口是否还在使用

优先看前端路由：

```bash
sed -n '1,260p' orin-frontend/src/router/routes.js
sed -n '1,260p' orin-frontend/src/router/index.js
```

### 看工作流是不是“真执行”

至少同时核对三层：

1. 前端节点配置和入口页
2. Java 侧 `workflow` 控制器和代理层
3. Python `orin-ai-engine/app/engine` 节点处理器

不要只因为页面可点开，就判断能力已经闭环。
