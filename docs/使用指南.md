# ORIN 使用指南

面向使用者：从哪里进入功能、联调时先看什么、如何快速验证。

## 1. 访问入口

| 服务 | 地址 |
|------|------|
| 前端管理台 | <http://localhost:5173> |
| 后端 API | <http://localhost:8080> |
| Swagger UI | <http://localhost:8080/swagger-ui/index.html> |
| OpenAPI JSON | <http://localhost:8080/v3/api-docs> |
| AI 引擎 | <http://localhost:8000> |
| 健康检查（后端） | `/v1/health`、`/api/v1/health` |
| 健康检查（AI 引擎） | `/health`、`/v1/health` |

启动方式见 [部署指南.md](./部署指南.md)。

## 2. 主导航

前端按当前 `orin-frontend/src/router/routes.js` 的 `SIDEBAR_MENU_CONFIG` 组织。判断“主入口”时，以侧边栏二级分组路径为准；`ROUTES` 常量里的落地页路径和历史重定向路径不能单独当作菜单入口。

当前显式二级分组入口共 10 条：

| 模块 | 分组 | 主路径 | 覆盖能力 |
|------|------|--------|----------|
| 智能体管理 | 智能体管理 | `/dashboard/agents/manage` | 列表、接入、控制台 |
| 智能体管理 | 会话管理 | `/dashboard/agents/session` | 会话记录、工作台、多智能体协同 |
| 智能体管理 | 扩展管理 | `/dashboard/agents/capability` | 技能、MCP 服务 |
| 运行监控 | 链路与分析 | `/dashboard/monitor/analysis` | Trace、Token、时延、错误统计 |
| 运行监控 | 运维操作 | `/dashboard/monitor/ops` | 日志归档、系统维护 |
| 系统设置 | 组织权限 | `/dashboard/system/organization` | 用户、部门、角色 |
| 系统设置 | 平台设置 | `/dashboard/system/platform` | 环境、通知、数据资产、MCP |
| 系统设置 | 模型与文件 | `/dashboard/system/resources` | 模型、定价 |
| 系统设置 | 网关与审计 | `/dashboard/system/security-ops` | 统一网关、监控配置、审计日志 |
| 系统设置 | 支持维护 | `/dashboard/system/support` | 帮助中心、统计分析 |

知识库管理、工作流中心、监控总览、告警与事件是侧边栏中的直接入口，不属于上述 10 个二级分组路径。顶部菜单由 `topMenuConfig.js` 通过角色过滤和开关控制可见性，当前不维护占位状态标记。

## 3. 常用功能入口

### 3.1 智能体与工作流

| 要做的事 | 从哪里进入 |
|----------|------------|
| 管理智能体、接入新智能体、打开控制台 | `/dashboard/agents/manage` |
| 查看会话、进入工作台、查看多智能体协同 | `/dashboard/agents/session` |
| 管理技能与 MCP 服务 | `/dashboard/agents/capability` |
| 管理和编排工作流 | 侧边栏“智能体管理 → 工作流中心” |

### 3.2 知识中心

| 要做的事 | 从哪里进入 |
|----------|------------|
| 检索知识库内容 | 侧边栏“知识库管理 → 知识检索” |
| 管理知识资产、文档、图谱相关资产 | 侧边栏“知识库管理 → 知识资产” |

### 3.3 监控运维

| 要做的事 | 从哪里进入 |
|----------|------------|
| 看监控总览、服务器状态、任务队列 | 侧边栏“运行监控 → 监控总览” |
| 查调用链路、Token、时延、错误统计 | `/dashboard/monitor/analysis` |
| 看告警与事件 | 侧边栏“运行监控 → 告警与事件” |
| 做日志归档和系统维护 | `/dashboard/monitor/ops` |

### 3.4 系统设置

| 要做的事 | 从哪里进入 |
|----------|------------|
| 管理用户、部门、角色 | `/dashboard/system/organization` |
| 管理环境、通知、数据资产、MCP 服务 | `/dashboard/system/platform` |
| 管理模型和定价 | `/dashboard/system/resources` |
| 管理统一网关、监控配置、审计日志 | `/dashboard/system/security-ops` |
| 查看帮助中心和统计分析 | `/dashboard/system/support` |

## 4. 第一次使用：最小验证流程

```bash
# 1. 启动三端
./manage.sh start

# 2. 烟测
bash scripts/smoke-test.sh

# 3. 浏览器访问
open http://localhost:5173
```

最小可验证清单：

- [ ] `/`、`/portal` 可打开，无白屏
- [ ] 登录成功（默认账号见 `application-dev.properties` 或种子脚本）
- [ ] `/dashboard/agents/manage` 可打开，智能体管理入口可见
- [ ] 侧边栏“智能体管理 → 工作流中心”可打开，工作流页面无白屏
- [ ] `/dashboard/system/security-ops` 可打开，统一网关与审计入口可见

## 5. 联调要点

### 5.1 看接口是否真存在

```bash
# 直接搜后端控制器
rg "@RequestMapping|@GetMapping|@PostMapping" \
   orin-backend/src/main/java/com/adlin/orin

# 或打开 Swagger
open http://localhost:8080/swagger-ui/index.html
```

### 5.2 看页面入口是否还在用

```bash
# 路由常量
sed -n '1,260p' orin-frontend/src/router/routes.js

# 顶部菜单可见性（角色过滤与开关）
sed -n '1,260p' orin-frontend/src/router/topMenuConfig.js

# 找开发中提示或未启用提示
rg "开发中|未启用|待实现" orin-frontend/src
```

### 5.3 看工作流/协作是否真执行

涉及工作流、协作、知识链路时，必须同时核对三层：

1. 前端节点配置和入口页
2. Java 后端控制器与代理层
3. Python `orin-ai-engine/app/engine` 节点处理器与 TaskRuntime

页面可点开 ≠ 能力闭环。

## 6. 文档使用原则

- 路由常量、菜单项、组件文件三者都存在，仍不等于能力闭环
- 部分路径（版本管理、协作配置等）是同页面别名复用
- 模块成熟度请对照 [功能完成度.md](./功能完成度.md)
- 接口字段以 Swagger 与控制器代码为准

## 7. 历史路径与重定向

以下路径来自 `LEGACY_ROUTE_REDIRECTS` 或旧文档，仅用于兼容访问和排查历史链接，不作为当前主导航入口：

| 旧路径示例 | 当前处理口径 |
|------------|--------------|
| `/dashboard/agents/list`、`/dashboard/agents/onboard`、`/dashboard/agents/chat-logs` | 重定向到智能体相关落地页，主入口从 `/dashboard/agents/manage` 或 `/dashboard/agents/session` 进入 |
| `/dashboard/applications/*` | 多数是当前落地页常量或旧别名；菜单分组入口以本指南第 2 节的 10 条路径为准 |
| `/dashboard/knowledge/*` | 历史知识库路径；从“知识库管理”侧边栏入口进入 |
| `/dashboard/resources/*` | 知识资源落地页常量；不作为二级分组主入口 |
| `/dashboard/runtime/*` | 运行监控落地页常量；分组入口从“监控总览”“链路与分析”“运维操作”进入 |
| `/dashboard/control/*` | 系统设置落地页常量；分组入口从 `/dashboard/system/organization`、`/dashboard/system/platform` 等进入 |
| `/dashboard/system/users`、`/dashboard/system/api-keys`、`/dashboard/system/audit-logs` | 旧系统管理路径；分别从组织权限、网关与审计等分组进入 |
