# ORIN 使用指南

面向使用者：从哪里进入功能、联调时先看什么、如何快速验证。

## 1. 访问入口

| 服务 | 地址 |
|------|------|
| 前端管理台 | <http://localhost:5173> |
| 后端 API | <http://localhost:8080> |
| Swagger UI | <http://localhost:8080/swagger-ui/index.html> |
| OpenAPI JSON | <http://localhost:8080/v3/api-docs> |
| AI 引擎 | <http://localhost:8000> |
| 健康检查（后端） | `/v1/health`、`/api/v1/health` |
| 健康检查（AI 引擎） | `/health`、`/v1/health` |

启动方式见 [部署指南.md](./部署指南.md)。

## 2. 主导航

前端按四大模块组织：

1. **智能体** — 智能体接入、对话、协作、工作流
2. **知识中心** — 知识库、检索实验室、向量/视觉实验室、图谱
3. **监控运维** — 监控大屏、Token/成本/时延/错误统计、Trace、任务队列
4. **系统管理** — 用户、角色、API Key、统一网关、模型配置、审计

> 路由结构以 `orin-frontend/src/router/routes.js` 为准。前端保留了部分历史路径的重定向（如 `/dashboard/knowledge/center` → `/dashboard/resources/center`），不要把别名当作独立能力。

## 4. 第一次使用：最小验证流程

```bash
# 1. 启动三端
./manage.sh start

# 2. 烟测
bash scripts/smoke-test.sh

# 3. 浏览器访问
open http://localhost:5173
```

最小可验证清单：

- [ ] `/`、`/portal` 可打开，无白屏
- [ ] 登录成功（默认账号见 `application-dev.properties` 或种子脚本）
- [ ] `/dashboard/applications/agents` 可打开，列表可加载
- [ ] `/dashboard/applications/workflows/visual` 首屏看到开始/结束节点
- [ ] `/dashboard/control/unified-api-docs` 在线调试对未配置 API Key 给出明确提示

## 5. 联调要点

### 5.1 看接口是否真存在

```bash
# 直接搜后端控制器
rg "@RequestMapping|@GetMapping|@PostMapping" \
   orin-backend/src/main/java/com/adlin/orin

# 或打开 Swagger
open http://localhost:8080/swagger-ui/index.html
```

### 5.2 看页面入口是否还在用

```bash
# 路由常量
sed -n '1,260p' orin-frontend/src/router/routes.js

# 顶部菜单状态（含 placeholder 标记）
sed -n '1,260p' orin-frontend/src/router/topMenuConfig.js

# 找占位/开发中标记
rg "status:\s*'placeholder'|开发中" orin-frontend/src
```

### 5.3 看工作流/协作是否真执行

涉及工作流、协作、知识链路时，必须同时核对三层：

1. 前端节点配置和入口页
2. Java 后端控制器与代理层
3. Python `orin-ai-engine/app/engine` 节点处理器与 TaskRuntime

页面可点开 ≠ 能力闭环。

## 6. 文档使用原则

- 路由常量、菜单项、组件文件三者都存在，仍不等于能力闭环
- 部分路径（版本管理、协作配置等）是同页面别名复用
- 模块成熟度请对照 [功能完成度.md](./功能完成度.md)
- 接口字段以 Swagger 与控制器代码为准
