# ORIN 使用指南

本文档面向当前仓库实现，重点说明“从哪里进入功能”和“联调时先看什么”。它只描述当前还能在代码里确认到的入口，不把“路由常量存在”“菜单项存在”“页面能打开”直接当成能力已闭环。

## 快速访问

- 前端开发地址：`http://localhost:5173`
- 后端地址：`http://localhost:8080`
- AI 引擎默认地址：`http://localhost:8000`
- OpenAPI / Swagger UI：`http://localhost:8080/swagger-ui/index.html`

## 推荐启动方式

### 一键启动

```bash
cd /path/to/ORIN
./manage.sh start
```

说明：

- `manage.sh` 会尝试启动 MySQL、后端、前端、Python AI 引擎
- 默认端口是前端 `5173`、后端 `8080`、AI 引擎 `8000`
- 脚本默认按本地开发场景编写，且更偏向 macOS + 本机 MySQL + AI 引擎 `venv`
- 如果环境不匹配，优先手动启动，更完整的环境配置见 [部署指南.md](/Users/adlin/Documents/Code/ORIN/docs/部署指南.md)

### 分别启动

```bash
cd orin-backend
mvn spring-boot:run
```

```bash
cd orin-ai-engine
venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000
```

```bash
cd orin-frontend
npm install
npm run dev
```

## 当前前端导航

当前主导航按模块分为四大块：

1. 智能体
2. 知识中心
3. 监控运维
4. 系统管理

如果你在旧文档里看到 `/dashboard/agent/*`、`/dashboard/workflow/*` 之类路径，不用惊讶。前端路由里保留了重定向兼容，但新的入口以 `orin-frontend/src/router/routes.js` 为准。

但要注意：

- `routes.js` 是路径常量，不等于所有路径都已经是稳定产品入口
- 实际挂载关系要看 `orin-frontend/src/router/index.js`
- 菜单暴露策略要看 `orin-frontend/src/router/topMenuConfig.js`
- 有些路径只是同一页面的别名或复用，不代表有独立能力闭环

## 常用功能入口

### 1. 智能体

当前更值得优先看的页面：

- 智能体列表：`/dashboard/agents/list`
- 智能体接入：`/dashboard/agents/onboard`
- 智能体控制台：`/dashboard/agents/console/:id`
- 会话记录：`/dashboard/agents/chat-logs`
- 协作任务：`/dashboard/agents/collaboration`
- 工作流入口：`/dashboard/agents/workflows`

边界说明：

- 接入外部 Agent 或模型服务
- 查看聊天记录和运行状态
- 调试协作任务或工作流执行
- `版本管理`、`测试与调试` 等路由目前仍复用列表页，不应当成独立成熟模块
- `协作任务列表`、`协作模式配置` 等路由当前也存在页面复用关系

### 2. 知识中心

当前更值得优先看的页面：

- 知识库列表：`/dashboard/knowledge/list`
- 新建知识库：`/dashboard/knowledge/create`
- 知识库详情：`/dashboard/knowledge/detail/:id`
- 检索实验室：`/dashboard/knowledge/retrieval-lab`
- 向量实验室：`/dashboard/knowledge/embedding-lab`
- 视觉实验室：`/dashboard/knowledge/vlm-lab`
- 知识图谱：`/dashboard/knowledge/graph`

现状说明：

- 文档上传、解析、分块、向量化相关能力在这部分收口
- `知识图谱` 路由已挂载，但顶部菜单配置仍把它标成 `placeholder`
- `同步管理` 目前复用的是系统侧同步页面，不代表知识中心已有完整独立同步工作台
- 联调前建议同时看页面、后端控制器和实际服务实现

### 3. 监控运维

当前更值得优先看的页面：

- 监控总览：`/dashboard/monitor/dashboard`
- Token 统计：`/dashboard/monitor/tokens`
- 成本统计：`/dashboard/monitor/costs`
- 时延分析：`/dashboard/monitor/latency`
- 错误统计：`/dashboard/monitor/errors`
- Trace 查看：`/dashboard/monitor/traces`
- 任务队列：`/dashboard/monitor/tasks`
- 系统维护：`/dashboard/monitor/maintenance`

适用场景：

- 观察调用链、成本、错误和任务堆积
- 排查 API 网关、工作流、知识处理链路的问题
- `告警规则` 当前复用的是告警管理页
- `通知渠道` 当前会重定向到系统设置
- `版本升级` 当前只是维护页别名，不是独立模块

### 4. 系统管理

当前更值得优先看的页面：

- 用户管理：`/dashboard/system/users`
- 角色管理：`/dashboard/system/roles`
- API Key 管理：`/dashboard/system/api-keys`
- 统一网关：`/dashboard/system/gateway`
- 模型配置：`/dashboard/system/models`
- 审计日志：`/dashboard/system/audit-logs`
- 系统维护：`/dashboard/system/maintenance`

边界说明：

- 这部分很多入口是管理员专用
- `settings/*` 是统一壳层，很多子页复用现有系统页面
- `external-frameworks/*`、`mcp-service` 等旧路径现在更多是重定向或兼容入口

## 常见联调方式

### 看接口是否存在

优先打开 Swagger：

```bash
open http://localhost:8080/swagger-ui/index.html
```

如果只是快速核对控制器入口，可以直接搜后端注解：

```bash
rg "@RequestMapping|@GetMapping|@PostMapping" orin-backend/src/main/java/com/adlin/orin
```

### 看页面入口是否还在使用

优先看前端路由和菜单配置：

```bash
sed -n '1,260p' orin-frontend/src/router/routes.js
sed -n '1,260p' orin-frontend/src/router/index.js
sed -n '1,260p' orin-frontend/src/router/topMenuConfig.js
```

建议额外检查：

```bash
rg -n "status:\\s*'placeholder'|开发中|ElMessage\\.(info|warning)\\(" orin-frontend/src
```

### 看工作流是不是“真执行”

至少同时核对三层：

1. 前端节点配置和入口页
2. Java 侧 `workflow` 控制器和代理层
3. Python `orin-ai-engine/app/engine` 节点处理器

不要只因为页面可点开，就判断能力已经闭环。

## 文档使用原则

- 路由常量、菜单项、组件文件三者都存在，仍然不等于能力已闭环。
- 组件名存在不等于它是当前唯一入口，例如某些版本管理、调试、协作配置路径仍存在页面复用。
- 历史报告里如果写“已完成”“90%+”，请再对照 [系统功能实现评估报告.md](/Users/adlin/Documents/Code/ORIN/docs/系统功能实现评估报告.md)。
- 涉及环境变量、数据库、端口和部署行为时，以 [部署指南.md](/Users/adlin/Documents/Code/ORIN/docs/部署指南.md) 和实际脚本为准。
