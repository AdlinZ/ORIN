# ORIN 使用指南

面向使用者：从哪里进入功能、联调时先看什么、如何快速验证。

## 1. 访问入口

| 服务 | 地址 |
|------|------|
| 前端管理台 | <http://localhost:5173> |
| 后端 API | <http://localhost:8080> |
| Swagger UI | <http://localhost:8080/swagger-ui/index.html> |
| OpenAPI JSON | <http://localhost:8080/v3/api-docs> |
| AI 引擎 | <http://localhost:8000> |

启动方式见 [部署指南.md](./部署指南.md)。

## 2. 主导航

前端按当前 `orin-frontend/src/router/topMenuConfig.js` 组织顶部菜单，并按角色过滤一级菜单和子菜单。判断“主入口”时，以角色可见菜单为准；`ROUTES` 常量里的落地页路径和历史重定向路径不能单独当作菜单入口。

当前顶部菜单主入口共 6 组：

| 模块 | 分组 | 主路径 | 覆盖能力 |
|------|------|--------|----------|
| 智能体管理 | 智能体管理 | `/dashboard/applications` | 工作台、列表、会话、多智能体协同、扩展管理 |
| 工作流管理 | 工作流管理 | `/dashboard/applications/workflows` | 工作流中心、可视化编排、执行记录 |
| 知识库管理 | 知识库管理 | `/dashboard/resources/center` | 知识检索、知识资产 |
| 运行监控 | 运行监控 | `/dashboard/runtime` | 监控总览、服务器、Trace、用量、性能、告警日志 |
| 组织权限 | 组织权限 | `/dashboard/control` | 用户、部门、角色 |
| 系统设置 | 系统设置 | `/dashboard/control` | 模型、定价、环境、通知、数据资产、MCP、统一网关 |

顶部菜单由 `topMenuConfig.js` 通过角色过滤和开关控制可见性。当前不维护占位状态标记；菜单是否可见不等于能力已经完整闭环。

## 2.1 角色入口

| 角色 | 默认入口 | 说明 |
|------|----------|------|
| `ROLE_SUPER_ADMIN` / `ROLE_ADMIN` | `/dashboard/runtime/overview` | 可见管理台完整入口 |
| `ROLE_PLATFORM_ADMIN` | `/dashboard/runtime/overview` | 可见运行监控与系统设置；组织权限默认不开启 |
| `ROLE_OPERATOR` | `/dashboard/applications/agents` | 可见智能体、工作流、知识库执行入口 |
| `ROLE_USER` | `/portal` | 使用服务门户，不进入管理台模块 |

角色边界详见 [角色矩阵.md](./角色矩阵.md)。直接输入不可见模块 URL 时，路由守卫会按同一角色矩阵拦截。

## 4. 第一次使用：最小验证流程

```bash
# 1. 启动三端
./manage.sh start

# 2. 烟测
bash scripts/smoke-test.sh

# 3. 浏览器访问
open http://localhost:5173
```

最小可验证清单：

- [ ] `/`、`/portal` 可打开，无白屏
- [ ] 登录成功（默认账号由本地配置或初始化向导创建；生产环境不得使用示例密码）
- [ ] `/dashboard/applications/agents` 可打开，智能体管理入口可见
- [ ] `/dashboard/applications/workflows` 可打开，工作流页面无白屏
- [ ] `/dashboard/control/gateway` 可打开，统一网关与 API Key 入口可见

## 5. 联调要点

### 5.1 看接口是否真存在

```bash
# 直接搜后端控制器
rg "@RequestMapping|@GetMapping|@PostMapping" \
   orin-backend/src/main/java/com/adlin/orin

# 或打开 Swagger
open http://localhost:8080/swagger-ui/index.html
```

### 5.2 看页面入口是否还在用

```bash
# 路由常量
sed -n '1,260p' orin-frontend/src/router/routes.js

# 顶部菜单可见性（角色过滤与开关）
sed -n '1,260p' orin-frontend/src/router/topMenuConfig.js

# 找开发中提示或未启用提示
rg "开发中|未启用|待实现" orin-frontend/src
```

### 5.3 看工作流/协作是否真执行

涉及工作流、协作、知识链路时，必须同时核对三层：

1. 前端节点配置和入口页
2. Java 后端控制器与代理层
3. Python `orin-ai-engine/app/engine` 节点处理器与 TaskRuntime

页面可点开 ≠ 能力闭环。

## 6. 文档使用原则

- 路由常量、菜单项、组件文件三者都存在，仍不等于能力闭环
- 部分路径（版本管理、协作配置等）是同页面别名复用
- 模块成熟度请对照 [功能完成度.md](./功能完成度.md)
- 接口字段以 Swagger 与控制器代码为准

## 7. 历史路径与重定向

以下路径来自 `LEGACY_ROUTE_REDIRECTS` 或旧文档，仅用于兼容访问和排查历史链接，不作为当前主导航入口：

| 旧路径示例 | 当前处理口径 |
|------------|--------------|
| `/dashboard/agents/list`、`/dashboard/agents/onboard`、`/dashboard/agents/chat-logs` | 重定向到智能体相关落地页，主入口从 `/dashboard/applications/agents`、`/dashboard/applications/workspace` 或 `/dashboard/applications/conversations` 进入 |
| `/dashboard/applications/*` | 当前智能体与工作流落地页常量；菜单入口以本指南第 2 节的顶部菜单为准 |
| `/dashboard/knowledge/*` | 历史知识库路径；当前从 `/dashboard/resources/center` 或 `/dashboard/resources/assets` 进入 |
| `/dashboard/resources/*` | 当前知识资源落地页常量；菜单入口以“知识库管理”为准 |
| `/dashboard/runtime/*` | 当前运行监控落地页常量；菜单入口以“运行监控”为准 |
| `/dashboard/control/*` | 当前系统设置与组织权限落地页常量；以 `topMenuConfig.js` 的组织权限、系统设置菜单为准 |
| `/dashboard/system/users`、`/dashboard/system/api-keys`、`/dashboard/system/audit-logs` | 旧系统管理路径；分别进入 `/dashboard/control/users`、`/dashboard/control/gateway`、`/dashboard/control/audit-logs` 等当前落地页 |
