# F1.4 测试失败排障说明

> 本文档帮助开发者在测试失败时快速定位问题根因。

## 快速索引

| 失败特征 | 优先检查 |
|---------|---------|
| `Cannot invoke xxx because "xxx" is null` | Mock 未注入或未 stub |
| `Bean creation exception` | Spring 上下文加载失败，检查 `@SpringBootTest` 依赖 |
| `Connection refused` / `Service unavailable` | 外部依赖未启动（Redis/Milvus/MySQL） |
| `NoSuchBeanDefinitionException` | Service 层实现类未加 `@Service` 或构造器参数缺失 |
| `MockCreationException` | Mock 类与方法签名不匹配，检查接口方法 |
| `H2 DB 相关错误` | 测试用 H2 内存库，检查 `application-test.properties` |
| `Redis connection refused` | 测试 profile 的 Redis 配置或 embedded redis |
| `Milvus 连接超时` | Milvus 未启动或网络不可达（测试环境常见） |

## 1. 按测试类型定位

### 1.1 Mockito 单元测试 (`@ExtendWith(MockitoExtension.class)`)

**典型文件**：`ErrorCodeTest`, `GlobalExceptionHandlerTest`, `AgentSmokeTest`

**失败原因**：

| 错误信息 | 根因 | 解决方法 |
|---------|------|--------|
| `MockitoException: Cannot mock/spy class Xxx` | 类是 final / private / interface | 用 `mock(Xxx.class)` 或检查类修饰符 |
| `Wanted but not invoked` | Mock 方法未被调用 | 检查调用链是否正确触发，或 `verify()` 时序 |
| `NullPointerException: Cannot invoke because "xxx" is null` | 未 `@Mock` 注入 | 确认 `@Mock` 注解存在且字段可见 |
| `WrongTypeOfReturnValue` | stub 返回类型与实际不匹配 | 检查 `when().thenReturn()` 类型一致性 |

**排障命令**：

```bash
cd orin-backend
# 只运行单元测试（无 Spring 上下文）
mvn test -Dtest=ErrorCodeTest,ResultTest,GlobalExceptionHandlerTest
```

### 1.2 Spring 集成测试 (`@SpringBootTest`)

**典型文件**：`KnowledgeGraphServiceTest`, `CollaborationServiceUnitTest`

**失败原因**：

| 错误信息 | 根因 | 解决方法 |
|---------|------|--------|
| `BeanCreationException` | 依赖 Bean 未定义或构造失败 | 检查 `application-test.properties` 配置 |
| `Failed to load ApplicationContext` | 上下文初始化失败 | 添加 `--debug` 查看详细堆栈 |
| `UnsatisfiedDependencyException` | 构造器参数 Bean 缺失 | 检查依赖服务是否在 test profile 中 mock |
| `FlywayException` | Flyway 迁移失败 | 在 `application-test.properties` 设置 `spring.flyway.enabled=false` |

**排障命令**：

```bash
# 开启 debug 模式
mvn test -Dtest=KnowledgeGraphServiceTest -Dspring.jpa.show-sql=true

# 查看详细上下文加载日志
mvn test -Dtest=CollaborationServiceUnitTest --debug
```

### 1.3 数据库相关测试

**检查项**：

1. `H2 内存库`：`spring.datasource.url=jdbc:h2:mem:testdb`
2. `DDL 策略`：`spring.jpa.hibernate.ddl-auto=create-drop`
3. `Flyway 禁用`：`spring.flyway.enabled=false`

**常见错误**：

| 错误信息 | 根因 | 解决方法 |
|---------|------|--------|
| `Table "XXX" not found` | H2 表未创建 | 检查 Entity 类注解和 `ddl-auto` 策略 |
| `Unique index violation` | 重复数据 | 测试前 `deleteAll()` 或用 `@Transactional` 回滚 |
| `Blob/Clob type error` | MySQL 与 H2 数据类型差异 | 使用 H2 兼容的类型，避免 MySQL 特有语法 |

## 2. 按模块定位

### 2.1 Agent 模块测试（`AgentSmokeTest`, `AgentFailureTest`）

**Mock 依赖**：

```java
@Mock private SiliconFlowIntegrationService siliconFlowIntegrationService;
@Mock private DifyIntegrationService difyIntegrationService;
@Mock private OllamaIntegrationService ollamaIntegrationService;
@Mock private AgentAccessProfileRepository accessProfileRepository;
```

**常见失败**：

| 场景 | 失败原因 | 排障 |
|-----|---------|-----|
| `onboardAgent()` 失败 | `siliconFlowIntegrationService.testConnection()` 未 stub | 检查 `when(...).thenReturn()` 是否在测试方法中设置 |
| `chat()` 返回 null | `siliconFlowAgentManageService.chat()` 未 stub | 确认 `viewType` 与实际调用路径匹配 |
| `Connection timeout` | 外部 provider 实际调用 | 确认测试方法中所有外部调用都已 Mock |

### 2.2 Knowledge 模块测试（`KnowledgeSmokeTest`, `RetrievalServiceTest`）

**依赖外部服务**：Milvus（向量数据库）

**常见失败**：

| 错误信息 | 根因 | 解决方法 |
|---------|------|--------|
| `Milvus connection refused` | Milvus 未启动 | `docker ps` 检查 Milvus 容器 |
| `Collection not found` | 向量集合不存在 | 测试前确认 Milvus 已初始化 |
| `ORA-xxxxx` | Oracle 驱动未找到 | 检查 `pom.xml` 是否包含 Oracle 驱动依赖 |

**降级方案**：使用 `OcrAsrDegradationTest` 的 Mock 模式，不依赖真实 Milvus。

### 2.3 Workflow 模块测试（`WorkflowServiceTest`, `GraphExecutorTest`）

**Mock 依赖**：外部 AI Provider 调用

**常见失败**：

| 错误信息 | 根因 | 解决方法 |
|---------|------|--------|
| `NullPointerException` 在 `GraphExecutor` | `nodeExecutor` 或 `workflowEngine` 未注入 | 检查 `@BeforeEach` 构造器注入 |
| `Execution timeout` | 工作流节点执行超时 | 检查 `WorkflowProperties` 超时配置 |

### 2.4 Collaboration 模块测试（`CollaborationServiceUnitTest`）

**Mock 依赖**：Workflow Engine, Agent Executor

**常见失败**：

| 错误信息 | 根因 | 解决方法 |
|---------|------|--------|
| `verify()` 断言失败 | 调用次数/参数不匹配 | 使用 `atLeastOnce()`, `atMost()` 放宽约束 |
| `Stream closed` | 并行任务资源未正确释放 | 检查 `@AfterEach` 清理逻辑 |

## 3. 外部依赖缺失的快速判断

### 3.1 Redis

```bash
# 检查 Redis 是否可用
redis-cli ping
# 期望返回：PONG

# 本地无 Redis 时的测试策略：
# 测试 profile 已配置使用 mock 或 embedded redis
# 如仍失败，检查 application-test.properties 中的 spring.data.redis.*
```

### 3.2 Milvus

```bash
# 检查 Milvus 容器
docker ps | grep milvus

# 如未启动
docker run -d --name milvus -p 19530:19530 milvusdb/milvus:latest
```

### 3.3 MySQL/PostgreSQL

```bash
# 检查数据库连接
mysql -u root -p -e "SELECT 1"

# H2 内存模式测试不需要外部数据库
# 如报 MySQL driver 缺失，检查是否在 test scope 引入了 mysql-connector-java
```

| 错误码 | 含义 | 关联测试失败 |
|-------|------|------------|
| `10000` | 系统内部错误 | 检查 `ErrorCodeTest` |
| `20001` | 资源不存在 | `GlobalExceptionHandlerTest` |
| `30001` | Agent 不存在 | `AgentSmokeTest` |
| `40001` | Knowledge 不存在 | `KnowledgeSmokeTest` |
| `60001` | Workflow 不存在 | `WorkflowServiceTest` |
| `70001` | 认证失败 | `GlobalExceptionHandlerTest` |
| `90001` | 参数校验失败 | `GlobalExceptionHandlerTest` |

## 5. 测试启动失败的标准排查流程

```
1. 检查 Java 版本
   java -version  # 期望 17+

2. 检查 Maven 依赖是否完整
   mvn dependency:tree -DincludeScope=test

3. 清理并重新编译
   mvn clean compile test-compile

4. 运行单个测试类
   mvn test -Dtest=<TestClassName>

5. 开启详细日志
   mvn test -Dtest=<TestClassName> -Dspring.jpa.show-sql=true -Dorg.slf4j.simpleLogger.defaultLogLevel=DEBUG
```

| 生成测试报告 | `mvn test surefire-report:report` |

## 7. 已知测试不稳定因素

| 编号 | 描述 | 临时方案 |
|-----|------|---------|
| T-001 | `TaskRetryTest` 在高负载下偶发超时 | 单独运行 `mvn test -Dtest=TaskRetryTest` |
| T-002 | `AgentBatchJobTest` 并发量大时 mock 失效 | 减少并发度或改用 `SynchronousExecutor` |
| T-003 | `OcrAsrDegradationTest` 依赖外部 OCR 服务 | 确保测试环境变量 `MOCK_OCR=true` |
