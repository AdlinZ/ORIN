# 并行协作端到端验收路径 (E3.3)

> 本文档记录并行协作（PARALLEL）的完整端到端验收路径，包括调用序列、输入输出和验收检查点。
> 基于 `CollaborationOrchestrator` + `CollaborationExecutor` 实现，并行能力依赖 `CompletableFuture` 异步执行。

---

## 并行协作核心流程

```
用户意图
    │
    ▼
createPackage(mode="PARALLEL")   创建协作包
    │
    ▼
decompose()                     任务分解（生成可并行的子任务，dependsOn=[]）
    │
    ▼
[并行执行] 同时触发多个 subtask 的异步执行
    │
    ├─ executor.executeSubtask(subtask_1) → CompletableFuture
    ├─ executor.executeSubtask(subtask_2) → CompletableFuture
    └─ executor.executeSubtask(subtask_3) → CompletableFuture
    │
    ▼
[等待] CompletableFuture.allOf(...) 等待所有并行任务完成
    │
    ▼
isAllSubtasksCompleted()        检查是否全部完成
    │
    ▼
complete()                      完成协作包
```

---

## 典型场景：ANALYSIS 类并行协作

**用户意图**：帮我分析这个问题并同时搜索相关资料

### 步骤 1：创建并行协作包

**请求**：
```java
CollaborationPackage pkg = orchestrator.createPackage(
    intent = "帮我分析这个问题并同时搜索相关资料",
    category = "ANALYSIS",
    priority = "NORMAL",
    complexity = "MEDIUM",
    collaborationMode = "PARALLEL",
    createdBy = "user_001",
    traceId = "trace_par__001"
);
```

**返回**：
```json
{
  "packageId": "par_001",
  "intent": "帮我分析这个问题并同时搜索相关资料",
  "intentTag": {
    "category": "ANALYSIS",
    "priority": "NORMAL",
    "complexity": "MEDIUM",
    "needReview": false,
    "needConsensus": false
  },
  "collaborationMode": "PARALLEL",
  "status": "PLANNING",
  "traceId": "trace_par_001"
}
```

### 步骤 2：任务分解

**请求**：
```java
CollaborationPackage result = orchestrator.decompose(
    packageId = "par_001",
    capabilities = ["analysis", "research"]
);
```

**分解结果**（ANALYSIS 类别）：
```json
{
  "packageId": "par_001",
  "status": "EXECUTING",
  "subTasks": [
    {
      "subTaskId": "1",
      "description": "收集相关信息和数据",
      "expectedRole": "SPECIALIST",
      "dependsOn": [],
      "confidence": 0.8,
      "status": "PENDING"
    },
    {
      "subTaskId": "2",
      "description": "分析数据并提取洞察",
      "expectedRole": "SPECIALIST",
      "dependsOn": ["1"],
      "confidence": 0.85,
      "status": "PENDING"
    },
    {
      "subTaskId": "3",
      "description": "总结分析结果",
      "expectedRole": "REVIEWER",
      "dependsOn": ["2"],
      "confidence": 0.9,
      "status": "PENDING"
    }
  ]
}
```

> **注意**：ANALYSIS 类别默认生成串行依赖链（2 依赖 1，3 依赖 2）。如果需要真正并行，应指定无依赖的子任务。

## 并行协作的关键：自定义无依赖分解

为了实现真正的并行执行，需要自定义子任务，确保它们之间没有依赖关系。

### 步骤 2'：手动创建并行子任务（无依赖）

**通过 Controller 创建后，手动添加无依赖子任务**：
```java
// 手动创建三个可并行的子任务（都没有依赖）
List<CollabSubtaskEntity> parallelTasks = List.of(
    CollabSubtaskEntity.builder()
        .subTaskId("A")
        .packageId("par_001")
        .description("搜索相关资料")
        .expectedRole("SPECIALIST")
        .dependsOn("[]")  // 无依赖
        .status("PENDING")
        .retryCount(0)
        .build(),
    CollabSubtaskEntity.builder()
        .subTaskId("B")
        .packageId("par_001")
        .description("整理现有数据")
        .expectedRole("SPECIALIST")
        .dependsOn("[]")  // 无依赖
        .status("PENDING")
        .retryCount(0)
        .build(),
    CollabSubtaskEntity.builder()
        .subTaskId("C")
        .packageId("par_001")
        .description("初步分析问题")
        .expectedRole("SPECIALIST")
        .dependsOn("[]")  // 无依赖
        .status("PENDING")
        .retryCount(0)
        .build()
);
```

### 步骤 3：并行执行（核心差异）

**获取可执行子任务**：
```java
List<CollabSubtaskEntity> executable = orchestrator.getExecutableSubtasks("par_001");
// 返回：A, B, C（三个都无依赖，都可立即执行）
```

**并行触发三个子任务**：
```java
// 三个异步调用同时触发（无等待）
CompletableFuture<String> futureA = executor.executeSubtask(taskA, "par_001", "trace_par_001");
CompletableFuture<String> futureB = executor.executeSubtask(taskB, "par_001", "trace_par_001");
CompletableFuture<String> futureC = executor.executeSubtask(taskC, "par_001", "trace_par_001");

// 等待所有并行任务完成
CompletableFuture.allOf(futureA, futureB, futureC).join();

// 获取结果
String resultA = futureA.get();  // "搜索到10篇相关资料..."
String resultB = futureB.get();  // "整理了20条数据记录..."
String resultC = futureC.get();  // "初步分析：问题可能出在..."
```

### 并行执行时序图

```
时间轴 →

[A] ──── RUNNING ──── COMPLETED ────
[B] ──── RUNNING ──── COMPLETED ────     并行执行，无等待
[C] ──── RUNNING ──── COMPLETED ────

vs 串行执行：

[A] ──── COMPLETED ────
[B] ──── COMPLETED ────                  串行执行，B等待A完成
[C] ──── COMPLETED ────
```

### 各子任务执行结果

| subTaskId | 描述 | 状态 | 结果 |
|-----------|------|------|------|
| A | 搜索相关资料 | COMPLETED | "搜索到10篇相关资料，包括技术文档和案例分析" |
| B | 整理现有数据 | COMPLETED | "整理了20条数据记录，发现3条异常数据" |
| C | 初步分析问题 | COMPLETED | "初步分析：问题可能出在数据源配置错误" |

### 步骤 4：检查并完成

```java
boolean allDone = orchestrator.isAllSubtasksCompleted("par_001");
// true

CollaborationPackage finalPkg = orchestrator.complete(
    packageId = "par_001",
    result = "并行分析完成：资料搜索10篇，数据整理20条，初步分析完成"
);
```

## 并行 vs 串行核心差异

| 维度 | 串行协作 | 并行协作 |
|------|---------|---------|
| 依赖关系 | 子任务有 `dependsOn` | 子任务无 `dependsOn`（或为空） |
| 执行时机 | 等待前置任务完成后执行 | 所有可执行任务同时启动 |
| 可执行任务数 | 同一时间只有 1 个 | 同一时间最多 `maxParallel` 个 |
| 执行时间 | T1 + T2 + T3 | max(T1, T2, T3) |
| 实现方式 | 循环调用 `getExecutableSubtasks` | `CompletableFuture.allOf` 并行触发 |

## 验收检查点

### 并行执行能力
- [ ] `dependsOn=[]` 的多个子任务可同时获取为可执行
- [ ] `executor.executeSubtask()` 可并行触发，互不阻塞
- [ ] `CompletableFuture.allOf()` 正确等待所有并行任务

### 状态一致性
- [ ] 并行执行时各子任务状态独立（不受其他任务影响）
- [ ] 一个子任务失败不影响其他并行任务
- [ ] 所有并行任务完成后才能 `complete()`

### 记忆共享
- [ ] 各子任务结果正确写入黑板（`memoryService.writeToBlackboard`）
- [ ] 最终结果包含所有并行任务的输出

### 事件发布
- [ ] 各子任务的 `SUBTASK_STARTED` 事件正确触发
- [ ] 各子任务的 `SUBTASK_COMPLETED` 事件正确触发
- [ ] 事件时序可能交错（并行特征）

### 异常处理
- [ ] 单个子任务失败不影响整体协作包
- [ ] 失败的任务可单独重试
- [ ] 所有任务失败时协作包状态应为 `FAILED`

## 附录：运行方式

### 手动触发并行执行

```
POST /api/v1/collaboration/packages
{
  "intent": "并行分析任务",
  "category": "ANALYSIS",
  "collaborationMode": "PARALLEL"
}

POST /api/v1/collaboration/packages/{packageId}/decompose

# 手动创建无依赖子任务（通过数据库或专用接口）

# 并行触发执行
POST /api/v1/collaboration/packages/{packageId}/subtasks/A/execute
POST /api/v1/collaboration/packages/{packageId}/subtasks/B/execute
POST /api/v1/collaboration/packages/{packageId}/subtasks/C/execute

# 检查完成状态
GET /api/v1/collaboration/packages/{packageId}/completed

# 完成协作包
POST /api/v1/collaboration/packages/{packageId}/complete
{
  "result": "所有并行任务完成"
}
```
