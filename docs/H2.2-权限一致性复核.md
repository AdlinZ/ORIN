## 三、后端权限控制

### 3.1 SecurityConfig (`SecurityConfig.java`)

```java
.authorizeHttpRequests(auth -> auth
    .requestMatchers("/api/v1/auth/**").permitAll()
    .requestMatchers("/api/v1/multimodal/files/*/download").permitAll()
    .requestMatchers("/api/v1/system/providers/**").permitAll()
    // API 网关端点 - 由 ApiKeyAuthInterceptor 进行 API 密钥校验
    .requestMatchers("/v1/**").permitAll()
    // Agent 管理端点 - 无认证要求
    .requestMatchers("/api/v1/agents/**").permitAll()
    .requestMatchers("/api/v1/model-config/**").permitAll()
    .requestMatchers("/api/v1/conversation-logs/**").permitAll()
    .requestMatchers("/api/v1/knowledge/diagnose/**").permitAll()
    // 用户权限管理端点 - 无认证要求
    .requestMatchers("/api/v1/departments/**").permitAll()
    .requestMatchers("/api/v1/roles/**").permitAll()
    .requestMatchers("/api/v1/users/**").permitAll()
    // 其他端点需要认证
    .requestMatchers("/api/v1/**").authenticated()
)
```

### 3.2 已知问题（来自 H2.1）

**严重**: `excludePathPatterns` 导致权限绕过

```java
// WebConfig.java 中的拦截器配置
registry.addInterceptor(apiKeyAuthInterceptor)
    .addPathPatterns("/v1/**", "/api/v1/**")
    .excludePathPatterns(
        "/api/v1/agents/**",        // 智能体管理
        "/api/v1/model-config/**",  // 模型配置
        "/api/v1/departments/**",   // 部门管理
        "/api/v1/roles/**",          // 角色管理
        "/api/v1/users/**"           // 用户管理
    );
```

**风险**: 这些端点被排除在 API Key 验证之外，如果请求也没有 JWT Token，则完全没有任何认证。

## 四、一致性问题汇总

### 4.1 系统管理模块

## 五、结论与建议

### 5.1 已确认问题

1. **系统管理端点** (`/api/v1/users/**`, `/api/v1/departments/**`, `/api/v1/roles/**`):
   - 前端要求 `ROLE_ADMIN`
   - 后端配置为 `permitAll()` + `excludePathPatterns`
   - **实际风险**: 任何人都可以访问这些端点

### 5.2 建议修复

1. **移除不必要的 `excludePathPatterns`**:
   ```java
   .excludePathPatterns(
       "/api/v1/agents/**",        // 保持或移除
       "/api/v1/departments/**",   // 移除，改为需要 JWT 认证
       "/api/v1/roles/**",          // 移除，改为需要 JWT 认证
       "/api/v1/users/**"           // 移除，改为需要 JWT 认证
   )
   ```

2. **统一前后端权限声明**:
   - 知识库和协作模块的前端路由应明确声明权限要求
   - 或在后端统一配置权限控制

### 5.3 下一步

建议按优先级修复：
1. **P0**: 移除 `/api/v1/users/**`, `/api/v1/departments/**`, `/api/v1/roles/**` 的 `excludePathPatterns`
2. **P1**: 统一知识库和协作模块的权限声明
3. **P2**: 完善 `@PreAuthorize` 方法级权限控制
