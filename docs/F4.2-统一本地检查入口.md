# F4.2: 统一本地检查入口

## 目标

提供一套统一的本地检查入口，让开发者在本地快速验证代码质量，无需分别记住各端的命令。

## 统一检查脚本

**路径**：`scripts/check-local.sh`

**执行方式**：

```bash
# 完整检查（三端全部）
bash scripts/check-local.sh

# 快速检查（仅 lint/build，无测试）
bash scripts/check-local.sh -q

# 仅后端
bash scripts/check-local.sh -b

# 仅前端
bash scripts/check-local.sh -f

# 仅 AI 引擎
bash scripts/check-local.sh -a
```

## 快速检查说明

### 后端（Java + Maven）

| 模式 | 命令 | 说明 |
|------|------|------|
| 编译 | `mvn compile` | 检查编译是否通过 |
| 快速lint | `mvn compile -q` | 静默编译检查 |
| 核心测试 | `mvn test -Dtest=ErrorCodeTest,ResultTest,BusinessExceptionTest,GlobalExceptionHandlerTest,KnowledgeGraphServiceTest,CollaborationServiceUnitTest,WorkflowServiceTest,AgentSmokeTest` | F1.x 核心回归测试 |
| 完整测试 | `mvn test` | 全部后端测试 |

### 前端（Vue 3 + Vite）

| 模式 | 命令 | 说明 |
|------|------|------|
| Lint 检查 | `npm run lint` | 代码规范检查 |
| 自动修复 | `npm run lint:fix` | 自动修复格式问题 |
| 构建检查 | `npm run build` | 验证构建是否通过 |

### AI 引擎（Python + FastAPI）

| 模式 | 命令 | 说明 |
|------|------|------|
| 语法检查 | `python3 -m py_compile app/main.py` | 快速语法验证 |
| 测试检查 | `python3 -m pytest tests/ -v` | 运行全部测试 |
| 指定测试 | `python3 -m pytest tests/test_llm_handler.py -v` | 运行指定测试文件 |

## 快速失败定位（按错误类型）

| 错误现象 | 检查顺序 |
|---------|---------|
| 后端编译失败 | `mvn compile` 看具体报错，通常是 import 缺包或类型错误 |
| 后端测试失败 | `mvn test -Dtest=<失败的类名>` 单独运行，快速定位 |
| 前端 lint 失败 | `npm run lint` 看具体文件和行号，`npm run lint:fix` 自动修复 |
| 前端构建失败 | `npm run build` 看报错，通常是依赖或环境变量问题 |
| AI 引擎报错 | `python3 -m pytest tests/ -v --tb=short` 看详细堆栈 |
| 整体环境问题 | `bash scripts/check-server-env.sh` 检查服务器环境 |

## 首次运行注意事项

1. **后端**：首次运行会下载 Maven 依赖，后续会缓存
2. **前端**：首次运行需 `npm install`，安装 node_modules
3. **AI 引擎**：依赖 Python 3.11+ 和 pytest，参考 `orin-ai-engine/README.md` 安装
