# 串行协作端到端验收路径 (E3.2)

> 本文档记录串行协作（SEQUENTIAL）的完整端到端验收路径，包括调用序列、输入输出和验收检查点。
> 基于 `CollaborationOrchestrator` + `CollaborationExecutor` 实现。

---

## 串行协作核心流程

```
用户意图
    │
    ▼
createPackage()     创建协作包
    │
    ▼
decompose()         任务分解（按 category 生成依赖链）
    │
    ▼
loop {
  getExecutableSubtasks()   获取可执行子任务（依赖已满足）
  │
  ▼
  executeSubtask()          执行子任务（调用 Agent）
  │
  ▼
  updateSubtaskStatus()     更新子任务状态
}
    │
    ▼
isAllSubtasksCompleted()    检查是否全部完成
    │
    ▼
complete()           完成协作包
```

---

## 典型场景：GENERATION 类串行协作

**用户意图**：帮我写一段代码并审查

### 步骤 1：创建协作包

**请求**：
```java
CollaborationPackage pkg = orchestrator.createPackage(
    intent = "帮我写一段Python代码并审查",
    category = "GENERATION",
    priority = "NORMAL",
    complexity = "MEDIUM",
    collaborationMode = "SEQUENTIAL",
    createdBy = "user_001",
    traceId = "trace_collab_001"
);
```

**返回**：
```json
{
  "packageId": "abc123def456",
  "intent": "帮我写一段Python代码并审查",
  "intentTag": {
    "category": "GENERATION",
    "priority": "NORMAL",
    "complexity": "MEDIUM",
    "needReview": false,
    "needConsensus": false
  },
  "collaborationMode": "SEQUENTIAL",
  "status": "PLANNING",
  "traceId": "trace_collab_001",
  "createdAt": "2026-03-28T10:00:00"
}
```

### 步骤 2：任务分解

**请求**：
```java
CollaborationPackage result = orchestrator.decompose(
    packageId = "abc123def456",
    capabilities = ["coding", "review"]
);
```

**分解结果**（GENERATION 类别生成 3 个子任务）：
```json
{
  "packageId": "abc123def456",
  "status": "EXECUTING",
  "subTasks": [
    {
      "subTaskId": "1",
      "description": "制定生成计划",
      "expectedRole": "PLANNER",
      "dependsOn": [],
      "confidence": 0.8,
      "status": "PENDING"
    },
    {
      "subTaskId": "2",
      "description": "执行生成任务",
      "expectedRole": "SPECIALIST",
      "dependsOn": ["1"],
      "confidence": 0.85,
      "status": "PENDING"
    },
    {
      "subTaskId": "3",
      "description": "审查生成结果",
      "expectedRole": "REVIEWER",
      "dependsOn": ["2"],
      "confidence": 0.9,
      "status": "PENDING"
    }
  ]
}
```

### 步骤 3：执行子任务循环

#### 第 1 轮：执行 subtask-1 (PLANNER)

**获取可执行子任务**：
```java
List<CollabSubtaskEntity> executable = orchestrator.getExecutableSubtasks("abc123def456");
// 返回：subtask "1"（无依赖，直接可执行）
```

**执行子任务**：
```java
// CollaborationExecutor.executeSubtask()
CompletableFuture<String> future = executor.executeSubtask(subtask, "abc123def456", "trace_collab_001");
String result = future.get();
// 调用 agentManageService.chat(agentId, "制定生成计划", null)
```

**Agent 返回**：
```
计划如下：
1. 确定代码功能需求
2. 编写Python代码
3. 添加单元测试
```

**更新子任务状态**：
```java
orchestrator.updateSubtaskStatus(
    packageId = "abc123def456",
    subTaskId = "1",
    status = "COMPLETED",
    result = "计划如下：1. 确定代码功能需求 2. 编写Python代码 3. 添加单元测试",
    errorMessage = null
);
```

#### 第 2 轮：执行 subtask-2 (SPECIALIST)

**获取可执行子任务**：
```java
List<CollabSubtaskEntity> executable = orchestrator.getExecutableSubtasks("abc123def456");
// 返回：subtask "2"（依赖 "1" 已完成）
```

**执行子任务**：
```java
String result = executor.executeSubtask(subtask, "abc123def456", "trace_collab_001").get();
// 调用 agentManageService.chat(agentId, "执行生成任务", null)
```

**Agent 返回**：
```python
def hello_world():
    print("Hello, World!")
    return True
```

**更新子任务状态**：
```java
orchestrator.updateSubtaskStatus(
    packageId = "abc123def456",
    subTaskId = "2",
    status = "COMPLETED",
    result = "def hello_world():\n    print(\"Hello, World!\")\n    return True",
    errorMessage = null
);
```

#### 第 3 轮：执行 subtask-3 (REVIEWER)

**获取可执行子任务**：
```java
List<CollabSubtaskEntity> executable = orchestrator.getExecutableSubtasks("abc123def456");
// 返回：subtask "3"（依赖 "2" 已完成）
```

**执行子任务**：
```java
String result = executor.executeSubtask(subtask, "abc123def456", "trace_collab_001").get();
// 调用 agentManageService.chat(agentId, "审查生成结果", null)
```

**Agent 返回**：
```
代码审查通过：
- 功能正常
- 建议添加类型注解
```

**更新子任务状态**：
```java
orchestrator.updateSubtaskStatus(
    packageId = "abc123def456",
    subTaskId = "3",
    status = "COMPLETED",
    result = "代码审查通过：功能正常，建议添加类型注解",
    errorMessage = null
);
```

### 步骤 4：检查并完成

**检查是否全部完成**：
```java
boolean allDone = orchestrator.isAllSubtasksCompleted("abc123def456");
// true
```

**完成协作包**：
```java
CollaborationPackage finalPkg = orchestrator.complete(
    packageId = "abc123def456",
    result = "任务完成：代码已生成并通过审查"
);
```

**最终结果**：

## 验收检查点

### 包级别
- [ ] `createPackage()` 成功创建包，状态为 `PLANNING`
- [ ] `decompose()` 正确根据 category 生成依赖链
- [ ] `complete()` 最终状态为 `COMPLETED`
- [ ] `traceId` 全程透传

### 子任务依赖
- [ ] 无依赖的子任务（`dependsOn=[]`）第一时间可执行
- [ ] 有依赖的子任务在前置任务完成前不可执行
- [ ] `getExecutableSubtasks()` 只返回依赖已满足的 PENDING 任务

### 子任务状态机
- [ ] 状态流转：`PENDING` → `RUNNING` → `COMPLETED`
- [ ] 失败状态：`PENDING` → `RUNNING` → `FAILED`
- [ ] 失败后可重试：`FAILED` → `PENDING`（`retryCount++`）

### 记忆共享
- [ ] 子任务结果写入黑板（`memoryService.writeToBlackboard`）
- [ ] 最终结果写入最终检查点

### 事件发布
- [ ] `PACKAGE_CREATED` 事件
- [ ] `PACKAGE_DECOMPOSED` 事件
- [ ] `SUBTASK_STARTED` 事件
- [ ] `SUBTASK_COMPLETED` 事件

## 附录：不同类别的子任务链

| Category | 子任务链（默认） |
|----------|-----------------|
| `ANALYSIS` / `RESEARCH` | 收集信息 → 分析洞察 → 总结 |
| `GENERATION` / `CODING` | 制定计划 → 执行生成 → 审查结果 |
| `REVIEW` | 执行待审内容 → 审查评估 → 确认结论 |
| `TESTING` | 执行测试 → 分析结果 → 修复问题 → 验证修复 |
| `GENERAL`（默认） | 理解需求 → 执行任务 → 审查结果 |

---

## 附录：状态转换表

### 协作包状态
```
PLANNING → DECOMPOSING → EXECUTING → COMPLETED
                          ↓
                       FAILED / FALLBACK
```

### 子任务状态
```
PENDING → RUNNING → COMPLETED
              ↓
           FAILED → PENDING（重试）或 SKIPPED
```

## 运行方式

```bash
cd orin-backend
./mvnw test -Dtest=CollaborationServiceTest
```

### 手动触发（通过 Controller）

```
POST /api/collaboration/package/create
{
  "intent": "帮我写一段Python代码并审查",
  "category": "GENERATION",
  "priority": "NORMAL",
  "complexity": "MEDIUM",
  "collaborationMode": "SEQUENTIAL"
}

POST /api/collaboration/package/{packageId}/decompose

POST /api/collaboration/package/{packageId}/subtask/{subTaskId}/execute

POST /api/collaboration/package/{packageId}/subtask/{subTaskId}/status
{
  "status": "COMPLETED",
  "result": "..."
}

POST /api/collaboration/package/{packageId}/complete
{
  "result": "任务完成"
}
```
