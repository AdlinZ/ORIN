# ORIN 前端商务 UI 规范

## 设计原则

- 页面背景默认使用纯白色，避免灰底页面、装饰图案底和大面积色块。
- 页面层级依靠布局、留白、边框、标题层级和表格密度表达，不依赖渐变、发光或强阴影。
- 毛玻璃只允许用于侧栏、顶栏、浮层等结构性容器，透明度和模糊强度必须克制。
- 新增页面不得引入 `premium-card`、`glass-panel`、发光 Logo、渐变按钮或霓虹 hover。

## 管理页结构

- 管理页必须使用 `OrinPageShell` 作为页头入口。
- 筛选和页面主操作必须放入 `OrinActionBar` 或 `OrinFilterBar`，确保窄屏可换行。
- 指标摘要使用 `OrinMetricStrip`，治理状态使用 `OrinStatusSummary`。
- 表格外壳使用 `OrinDataTable`，表格操作列固定在右侧，行高保持紧凑。
- 分页放在表格底部或 `OrinDataTable` 的 footer 区域，避免散落在页面外。

## 状态与详情

- 空状态必须使用 `OrinEmptyState` 或 `OrinAsyncState`，文案要说明下一步动作。
- 禁止只写“暂无数据”；应写成“暂无模型资源，请先接入服务商模型”这类业务语义。
- 错误状态必须提供重试入口，加载状态优先局部加载表格或面板。
- 列表详情优先使用右侧详情面板或抽屉；创建和编辑使用表单弹窗。

## 例外

- 图表、进度条、状态标签可以保留必要的语义色。
- Playground 和可视化工作流编辑器允许保留编辑器内部必要的画布/节点状态样式，但入口页必须遵守本规范。
- 历史页面迁移时先保证主导航可直接访问页面符合规范，再处理低频内部页。
