# ORIN 可验收能力清单

本清单只记录本地浏览器或 smoke 路径可以复核的能力，不把预留页面等同于完成能力。

## 已完成

| 能力 | 验收路径 | 通过口径 |
| --- | --- | --- |
| 前端入口 | `/`, `/portal` | 页面可打开，无白屏 |
| 后端健康检查 | `GET /v1/health`, `GET /api/v1/health` | 无需 API Key，返回服务状态 |
| AI Engine 健康检查 | `GET /health`, `GET /v1/health` | 返回 `status`, `service`, `dependencies` |
| 旧知识中心路由 | `/dashboard/knowledge/center` | 自动进入 `/dashboard/resources/center` 兼容链路 |
| 统一 API 鉴权提示 | `/dashboard/control/unified-api-docs` | 在线调试明确提示 API Key 要求 |
| 工作流可视化首屏 | `/dashboard/applications/workflows/visual` | 新建或加载后开始/结束节点自动 fit view |

## 可联调

| 能力 | 验收路径 | 依赖 |
| --- | --- | --- |
| 知识库检索 | `/dashboard/resources/retrieval` | MySQL、Milvus、Embedding Provider、至少一个已入库文档 |
| 工作流执行 | `/dashboard/applications/workflows` -> 测试/执行记录 | 后端工作流管理接口、AI Engine 执行接口 |
| 智能体对话与 trace | `/dashboard/applications/agents` -> 控制台 | 有可用模型 provider 与 API Key |
| 统一网关统计 | `/dashboard/control/gateway` | 网关路由真实请求流量、Redis |
| MCP 连接测试 | `/dashboard/control/mcp-service` | 本机存在对应 STDIO 命令或 SSE 地址可访问 |

## 预留中

| 能力 | 当前口径 |
| --- | --- |
| MCP 工具市场完整工具列表 | 模板市场已展示，真实工具列表仍依赖具体服务接入 |
| 外部 provider 全量可用性 | 不默认可用，缺 API Key/模型/provider 时按统一错误语义返回 |
| RabbitMQ 协作队列 | 本地默认不强制启动；健康检查暴露 `rabbitmq` 依赖状态 |

## 最小命令

```bash
bash scripts/smoke-test.sh
```

已启动三端后可额外检查：

```bash
curl -fsS http://127.0.0.1:8080/v1/health
curl -fsS http://127.0.0.1:8080/api/v1/health
curl -fsS http://127.0.0.1:8000/health
curl -fsS http://127.0.0.1:8000/v1/health
curl -i http://127.0.0.1:8080/v1/models
```
