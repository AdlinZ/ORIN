## 当前已知基准

### 协作任务执行

| 操作 | 基准耗时 | 说明 |
|------|----------|------|
| 协作任务启动 | <100ms | 不含 Agent 调度 |
| 子任务分发 | <50ms | 串行模式 |
| 子任务分发 | <100ms | 并行模式 |
| 结果汇总 | <100ms | 4 个子任务以内 |

### 知识库操作

| 操作 | 基准耗时 | 说明 |
|------|----------|------|
| 向量检索 (1000条) | <200ms | Milvus |
| 向量检索 (10000条) | <500ms | Milvus |
| 文档解析 (PDF 1MB) | 3-10s | 视内容复杂度 |
| 分块处理 | <1s | 100 页以内 |

### AI 引擎

| 操作 | 基准耗时 | 说明 |
|------|----------|------|
| LLM 调用 (流式) | 首次 <3s | 含网络延迟 |
| LLM 调用 (非流式) | 首次 <5s | 含网络延迟 |
| Tool 执行 | <2s | 简单工具 |
| 多模型切换 | <1s | 无缝 |

## 高耗时风险点

### 需要关注的场景

| 场景 | 风险等级 | 原因 |
|------|----------|------|
| 大文档解析 (>10MB) | 🔴 高 | 内存/超时 |
| 向量检索 (>10000条) | 🟡 中 | 检索耗时 |
| 多 Agent 协作 (>5个) | 🟡 中 | 串行等待 |
| 图谱构建 (>5000节点) | 🔴 高 | 可能 OOM |
| 外部 API 调用 | 🟡 中 | 网络不稳定 |

## 性能优化建议

1. **知识库检索**: 考虑分页/游标，避免全量加载
2. **大文档**: 分片上传，异步解析
3. **图谱构建**: 增量构建，控制单次规模
4. **外部 API**: 添加熔断，超时降级
