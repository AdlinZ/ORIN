# ORIN 开发任务读取提示词

> 之后的 Claude，请阅读 `docs/任务分段书.md` 来了解待开发任务。

## 使用方法

1. **阅读任务分段书** - 查看 `docs/任务分段书.md` 获取完整任务列表
2. **选择任务** - 从对应阶段选择标记为 `[ ]` 的任务
3. **开始开发** - 单次对话应完成一个任务（如 A1.1, B2.3 等）

## 任务粒度说明

- **小任务（A1.1, A2.1 等）**: 单次对话可完成
- **中任务（A1, B1 等）**: 需要多次对话，可拆分为多个小任务
- **阶段（A, B, C, D）**: 2-8周的中长期目标

## 推荐开发顺序

按优先级依次进行：

1. **阶段 A**（2-3周）- 真相对齐与基础稳定化
2. **阶段 B**（4-6周）- 主链路闭环
3. **阶段 C**（4-6周）- 协作闭环与统一观测
4. **阶段 D**（4-8周）- 高级能力与平台化扩展

## 任务选择建议

- 优先选择 P0 级别的任务
- 每个任务完成后更新任务分段书中的状态 `[ ]` → `[x]`
- 如需修改任务描述或拆分任务，直接编辑 `docs/任务分段书.md`
