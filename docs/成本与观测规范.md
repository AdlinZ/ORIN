# 成本与观测字段规范

> G3.1: 统一 token、成本、trace、审计的字段命名  
> G3.2: 区分"请求成功但结果降级"和"请求失败"的观测语义  
> G3.3: 给多 provider 调用增加最小成本统计口径  
> G3.4: 输出一版成本与观测字段规范

---

## 字段命名规范

| 概念 | 字段名 | 类型 | 说明 |
|------|--------|------|------|
| 输入 token 数 | `input_tokens` | Integer | Prompt tokens |
| 输出 token 数 | `output_tokens` | Integer | Completion tokens |
| 总 token 数 | `total_tokens` | Integer | input + output |
| 耗时(毫秒) | `latency_ms` | Long | 请求耗时 |
| 成本(美元) | `cost` | Double | 实际费用 |
| Trace ID | `trace_id` | String | 全链路追踪 |
| 请求状态 | `status` | Enum | SUCCESS/DEGRADED/FAILED |

---

## 观测语义区分

| 场景 | status | cost | latency_ms | 说明 |
|------|--------|------|------------|------|
| 正常成功 | `SUCCESS` | 正常 | 正常 | - |
| 降级成功 | `DEGRADED` | 正常 | 偏高 | 如 fallback model |
| 请求失败 | `FAILED` | 0 | 有值 | 如超时/限流 |
