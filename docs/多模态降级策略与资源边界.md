# 多模态处理降级策略与资源边界

## 文档说明

本文档定义 OCR、ASR 等多模态处理的失败降级策略和资源消耗边界。

---

## 1. 概述

多模态处理涉及高成本、高延迟的 AI 推理操作，需要明确的降级策略确保系统稳定性。

### 1.1 支持的处理类型

| 类型 | 本地方案 | 云服务方案 |
|------|---------|-----------|
| OCR 图片文字识别 | Tesseract | SiliconFlow VLM (Qwen2-VL) |
| ASR 音频语音转写 | Whisper CLI | SiliconFlow ASR (Whisper) |

---

## 2. 降级策略

### 2.1 云服务降级链路

```
Primary: SiliconFlow Cloud (OCR/ASR)
    ↓ 不可用或超时
Fallback: 本地 Tesseract / Whisper CLI
    ↓ 不可用或超时
Final: 返回占位文本，标记人工处理
```

### 2.2 每种处理类型的降级策略

#### OCR（图片文字识别）

| 层级 | 条件 | 结果 | 标记 |
|------|------|------|------|
| L1 | SiliconFlow API 可用 | 返回识别文本 | `provider: siliconflow_vlm` |
| L2 | L1 失败 + Tesseract 可用 | 返回识别文本 | `provider: tesseract` |
| L3 | L1+L2 失败 | 返回占位文本 | `ocr_required: true` |
| L4 | L1+L2+L3 失败（IO异常） | 返回错误文本 | `error: <message>` |

#### ASR（音频语音转写）

| 层级 | 条件 | 结果 | 标记 |
|------|------|------|------|
| L1 | SiliconFlow ASR 可用 | 返回转写文本 | `provider: siliconflow_asr` |
| L2 | L1 失败 + Whisper CLI 可用 | 返回转写文本 | `provider: whisper_cli` |
| L3 | L1+L2 失败 | 返回占位文本 | `asr_required: true` |
| L4 | L1+L2+L3 失败（超时/IO异常） | 返回错误文本 | `error: <message>` |

### 2.3 超时配置

| 处理类型 | 云服务超时 | 本地进程超时 | 单位 |
|---------|-----------|-------------|------|
| OCR (SiliconFlow VLM) | 30 秒 | N/A | 秒 |
| ASR (SiliconFlow ASR) | 60 秒 | 300 秒 | 秒 |
| Tesseract | N/A | 30 秒 | 秒 |
| Whisper CLI | N/A | 300 秒（5分钟） | 秒 |

## 3. 资源消耗边界

### 3.1 图片处理限制

| 参数 | 值 | 说明 |
|------|-----|------|
| 单图最大尺寸 | 10 MB | 超过返回错误 |
| 支持格式 | jpg, jpeg, png, gif, bmp, webp, tiff, tif | - |
| Base64 编码后最大长度 | ~14 MB | data URI 格式 |
| 默认模型 | Qwen/Qwen2-VL-7B-Instruct | 可配置 |
| max_tokens | 4096 | 响应最大 token |

### 3.2 音频处理限制

| 参数 | 值 | 说明 |
|------|-----|------|
| 单音频最大时长 | 30 分钟 | 超过建议分段 |
| 单音频最大尺寸 | 50 MB | 超过返回错误 |
| 支持格式 | mp3, wav, m4a, aac, ogg, flac, wma, aiff | - |
| 默认模型 | openai/whisper-large-v3-turbo | 可配置 |

### 3.3 并发限制

| 资源 | 限制 | 说明 |
|------|------|------|
| 同时进行 OCR 请求 | 5 | SiliconFlow API 限流 |
| 同时进行 ASR 请求 | 3 | SiliconFlow ASR 限流 |
| Tesseract 进程池 | 2 | 本地 CPU 限制 |
| Whisper 进程池 | 1 | 本地 GPU 限制 |

## 4. 错误码与错误信息

### 4.1 OCR 错误码

| 错误码前缀 | 含义 | 处理建议 |
|-----------|------|---------|
| `[OCR Error] SiliconFlow API key not configured` | 未配置 API Key | 检查 siliconflow.api.key 配置 |
| `[OCR Error] Unexpected response from SiliconFlow` | API 响应异常 | 查看日志，重试 |
| `[OCR Unavailable]` | Tesseract 不可用 | 安装 Tesseract 或配置云服务 |
| `[OCR IO Error]` | 文件读取失败 | 检查文件路径和权限 |
| `[OCR Timeout]` | 处理超时 | 降低图片分辨率后重试 |
| `[No Text Detected]` | 图片无文字 | 确认图片内容 |

### 4.2 ASR 错误码

| 错误码前缀 | 含义 | 处理建议 |
|-----------|------|---------|
| `[ASR Error] SiliconFlow API key not configured` | 未配置 API Key | 检查 siliconflow.api.key 配置 |
| `[ASR Error] Unexpected response from SiliconFlow` | API 响应异常 | 查看日志，重试 |
| `[ASR Unavailable]` | Whisper 不可用 | 安装 Whisper 或配置云服务 |
| `[ASR IO Error]` | 文件读取失败 | 检查文件路径和权限 |
| `[ASR Timeout]` | 处理超时 | 缩短音频时长后重试 |
| `[No Speech Detected]` | 音频无语音 | 确认音频内容 |

## 5. 配置项

### 5.1 application.yml 配置

```yaml
siliconflow:
  api:
    key: ${SILICONFLOW_API_KEY:}
    base-url: https://api.siliconflow.cn/v1

ocr:
  provider: cloud                    # cloud / local
  model: Qwen/Qwen2-VL-7B-Instruct   # VLM 模型
  language: chi_sim+eng              # Tesseract 语言
  tessdata-path: /usr/local/share/tessdata  # Tesseract 数据路径

asr:
  provider: cloud                    # cloud / local
  model: openai/whisper-large-v3-turbo  # ASR 模型
  whisper-path: /usr/local/bin/whisper  # Whisper CLI 路径
```

## 6. 监控与告警

### 6.1 触发告警的条件

| 告警类型 | 条件 | 严重程度 |
|---------|------|---------|
| 云服务连续失败 | 3 次连续 OCR/ASR 调用失败 | WARNING |
| 资源超限 | 图片超过 10MB 或音频超过 50MB | WARNING |
| 处理超时 | 单次处理超过超时阈值的 2 倍 | WARNING |
| 云服务不可用 | API Key 未配置但使用了云服务 | CRITICAL |

### 6.2 审计日志字段

每个多模态处理任务记录以下字段：

- `traceId`: 链路追踪 ID
- `assetId`: 资产 ID
- `assetType`: MULTIMODAL_FILE
- `taskType`: CAPTIONING / TRANSCRIPTION
- `provider`: siliconflow_vlm / tesseract / siliconflow_asr / whisper_cli
- `charsExtracted`: 提取的字符数
- `processingTimeMs`: 处理耗时
- `status`: SUCCESS / FAILED
- `errorMessage`: 错误信息（如有）
