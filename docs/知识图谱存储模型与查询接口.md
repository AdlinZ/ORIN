# ORIN 知识图谱存储模型与查询接口

本文档定义知识图谱的存储模型和查询接口。

---

## 1. 当前存储模型（基于 JPA/SQL）

### 1.1 实体表：graph_entities

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| graph_id | VARCHAR(100) | 所属图谱ID |
| name | VARCHAR(500) | 实体名称 |
| entity_type | VARCHAR(100) | 实体类型（人物/地点/机构/概念） |
| description | TEXT | 实体描述 |
| source_document_id | VARCHAR(100) | 来源文档ID |
| source_chunk_id | VARCHAR(100) | 来源chunk ID |
| properties | TEXT | 额外属性（JSON） |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 1.2 关系表：graph_relations

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| graph_id | VARCHAR(100) | 所属图谱ID |
| source_entity_id | VARCHAR(100) | 源实体ID |
| target_entity_id | VARCHAR(100) | 目标实体ID |
| relation_type | VARCHAR(100) | 关系类型（位于/工作于/毕业于） |
| description | TEXT | 关系描述 |
| source_document_id | VARCHAR(100) | 来源文档ID |
| source_chunk_id | VARCHAR(100) | 来源chunk ID |
| properties | TEXT | 额外属性（JSON） |
| weight | DOUBLE | 权重/置信度 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 1.3 图谱表：knowledge_graphs

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| name | VARCHAR(200) | 图谱名称 |
| description | TEXT | 图谱描述 |
| build_status | VARCHAR(50) | 构建状态 |
| entity_count | INT | 实体数量 |
| relation_count | INT | 关系数量 |
| last_build_at | DATETIME | 最后构建时间 |
| last_success_build_at | DATETIME | 最后成功构建时间 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

## 2. 未来存储模型（Neo4j）

### 2.1 节点类型 (:Entity)

```cypher
(:Entity {
    id: String,
    name: String,
    entityType: String,
    description: String,
    sourceDocumentId: String,
    sourceChunkId: String,
    properties: Map,
    createdAt: DateTime
})
```

### 2.2 关系类型 (:RELATES_TO)

```cypher
(:Entity)-[:RELATES_TO {
    relationType: String,
    description: String,
    weight: Float,
    sourceDocumentId: String,
    sourceChunkId: String,
    properties: Map,
    createdAt: DateTime
}]->(:Entity)
```

### 2.3 索引

```cypher
CREATE INDEX entity_name IF NOT EXISTS FOR (e:Entity) ON (e.name);
CREATE INDEX entity_type IF NOT EXISTS FOR (e:Entity) ON (e.entityType);
CREATE INDEX relation_type IF NOT EXISTS FOR ()-[r:RELATES_TO]-() ON (r.relationType);
```

## 3. 查询接口定义

### 3.1 实体查询

| 接口 | 说明 | SQL版本 | Neo4j版本 |
|------|------|---------|-----------|
| 按ID查询实体 | GET /api/v1/graphs/{graphId}/entities/{entityId} | JPA findById | MATCH (e:Entity {id: $id}) RETURN e |
| 按类型查询实体 | GET /api/v1/graphs/{graphId}/entities?type={type} | JPA findByGraphIdAndEntityType | MATCH (e:Entity {graphId: $graphId, entityType: $type}) RETURN e |
| 查询实体的邻居 | GET /api/v1/graphs/{graphId}/entities/{entityId}/neighbors | 复杂JOIN查询 | MATCH (e:Entity {id: $id})-[r]-(neighbor) RETURN neighbor, r |
| 搜索实体 | GET /api/v1/graphs/{graphId}/entities/search?q={keyword} | LIKE查询 | MATCH (e:Entity) WHERE e.name CONTAINS $keyword RETURN e |

### 3.2 关系查询

| 接口 | 说明 | SQL版本 | Neo4j版本 |
|------|------|---------|-----------|
| 查询两个实体间的关系 | GET /api/v1/graphs/{graphId}/relations?from={entityId}&to={entityId} | WHERE查询 | MATCH (s)-[r]->(t) WHERE s.id=$from AND t.id=$to RETURN r |
| 按关系类型查询 | GET /api/v1/graphs/{graphId}/relations?type={type} | findByGraphIdAndRelationType | MATCH ()-[r:RELATES_TO {relationType: $type}]->() RETURN r |
| 查询路径 | GET /api/v1/graphs/{graphId}/paths?from={entityId}&to={entityId}&maxDepth=3 | 不支持 | MATCH path = shortestPath((s)-[*1..3]-(t)) WHERE s.id=$from AND t.id=$to RETURN path |

### 3.3 图谱查询

| 接口 | 说明 |
|------|------|
| 获取图谱概览 | GET /api/v1/graphs/{graphId}/overview - 返回实体数、关系数、状态 |
| 获取图谱统计 | GET /api/v1/graphs/{graphId}/stats - 返回各类型实体/关系数量分布 |

## 4. 服务层接口

```java
public interface GraphQueryService {
    // 实体操作
    Optional<GraphEntity> getEntity(String entityId);
    List<GraphEntity> getEntitiesByType(String graphId, String entityType);
    List<GraphEntity> getEntityNeighbors(String entityId);
    List<GraphEntity> searchEntities(String graphId, String keyword);

    // 关系操作
    Optional<GraphRelation> getRelation(String relationId);
    List<GraphRelation> getRelationsByType(String graphId, String relationType);
    List<GraphRelation> getPath(String graphId, String fromEntityId, String toEntityId, int maxDepth);

    // 图谱操作
    Optional<KnowledgeGraph> getGraph(String graphId);
    GraphOverview getGraphOverview(String graphId);
    GraphStats getGraphStats(String graphId);
}
```

## 5. 转换计划

当前实现使用 JPA 实体，存储在关系型数据库。当需要切换到 Neo4j 时：

1. 添加 Neo4j 依赖
2. 创建 `Neo4jGraphQueryService` 实现 `GraphQueryService` 接口
3. 使用 `@Primary` 注解标记 Neo4j 实现
4. JPA 实现作为备用

```java
// 切换时使用
// @Primary
// @Service
// public class Neo4jGraphQueryService implements GraphQueryService { ... }
```
