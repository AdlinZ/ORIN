## 文件上传限制

### 当前配置值

| 类型 | 配置项 | 默认值 | 最大值 | 备注 |
|------|--------|--------|--------|------|
| 单文件大小 | `spring.servlet.multipart.max-file-size` | 10MB | 可调 | Base64/图片 |
| 请求总大小 | `spring.servlet.multipart.max-request-size` | 12MB | - | 单次请求总和 |
| HTTP POST | `server.tomcat.max-http-post-size` | 100MB | - | Tomcat 级 |

### 超限行为

- **小于限制**: 正常上传
- **大于限制**: 返回 `413 Payload Too Large`

### 建议调整场景

| 场景 | 建议值 |
|------|--------|
| 纯文档上传 | 50MB |
| 含图片/视频 | 200MB |
| 大模型上下文 | 100MB |

## 知识图谱构建限制

| 资源项 | 默认值 | 说明 |
|--------|--------|------|
| 单次构建节点数 | 无硬限 | 建议 <10000 |
| 单次构建边数 | 无硬限 | 建议 <50000 |
| 单节点属性数 | 无硬限 | 建议 <100 |
| 构建超时 | 无硬限 | 建议 <30min |

### 超限行为

- **大数据量**: 可能导致内存溢出 (`OOM`)
- **超时**: 任务自动取消，需手动重试

|------|--------|----------|
| 文件过大 | 413 | "文件大小超过限制(10MB)，请压缩后重试" |

| 资源不足 | 503 | "系统资源不足，请稍后重试" |

## 熔断/限流配置

当前使用 `resilience4j` 实现：

```yaml
resilience4j:
  circuitbreaker:
    instances:
      siliconFlow:
        registerHealthIndicator: true
        slidingWindowSize: 10
        permittedNumberOfCallsInHalfOpenState: 3
        slidingWindowType: COUNT_BASED
        minimumNumberOfCalls: 5
        waitDurationInOpenState: 5s
        failureRateThreshold: 50
        eventConsumerBufferSize: 10
```
