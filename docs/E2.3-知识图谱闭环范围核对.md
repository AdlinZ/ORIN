#### 2.2.1 🚨 路由 Bug（严重）

```js
// KnowledgeGraphList.vue:173
const openGraphDetail = (graph) => {
  router.push(`/knowledge/graphs/${graph.id}`)  // ❌ 错误
}
```

**问题**：导航目标 `/knowledge/graphs/${graph.id}` 不存在于路由表中。
**正确路径**应为 `/dashboard/knowledge/graph/${graph.id}`。

**影响**：用户点击任意图谱卡片，页面不会跳转，404 也不会触发（Vue Router 的默认行为是静默失败或匹配到父级 redirect）。

**修复方式**：
```js
router.push(`/dashboard/knowledge/graph/${graph.id}`)
```

## 三、核心功能闭环分析

### 3.1 已真闭环 ✅

- **列表查询**：后端 `findAll` → 前端卡片展示，链路完整
- **详情查询**：后端 `findById` → 前端详情卡片，链路完整
- **实体/关系分页查询**：后端 `Page` → 前端表格展示，完整
- **搜索实体**：后端模糊查询 → 前端下拉/过滤，完整
- **CRUD**：创建、编辑、删除均已实现并验证

### 3.2 伪闭环 ⚠️

**触发构建 `POST /{graphId}/build`**

现状：`triggerBuild()` 只做一件事：
```java
graph.setBuildStatus(GraphBuildState.BUILDING);
return knowledgeGraphRepository.save(graph);
```

问题：
1. 没有异步任务，没有 worker，没有实际的实体/关系抽取
2. 没有任务 ID 返回，前端无法轮询构建进度
3. 状态永远是 `BUILDING`（除非手动改回），没有完成/失败回调
4. 没有任何外部服务调用（Neo4j、LLM 等均未集成）

**结论**：构建按钮点了"成功"（接口返回 200），但实际什么都没做。

### 3.3 完全未实现 ❌

- 构建任务的异步执行（无 JMQ/线程池任务投递）
- 实体抽取（Entity Extraction）—— 无 NLP/LLM 调用
- 关系抽取（Relation Extraction）—— 无知识图谱写入
- 构建状态轮询 / WebSocket 推送
- 与 Neo4j 或其他图数据库的集成
