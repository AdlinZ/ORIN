# F4.1: 三端最小检查命令

## 一、后端 (orin-backend)

### 技术栈
- Java 17 + Spring Boot 3.2.1
- Maven

### 最小检查命令

```bash
cd orin-backend

# 编译
mvn compile

# 运行测试
mvn test

# 运行特定测试类
mvn test -Dtest=AgentSmokeTest

# 打包
mvn package -DskipTests
```

### 常用检查组合

```bash
# 快速检查（仅编译）
mvn compile

# 完整检查（编译 + 测试）
mvn verify

# 带覆盖率检查
mvn test jacoco:report
```

---

## 二、前端 (orin-frontend)

### 技术栈
- Vue 3 + Vite 7
- Node.js

### 最小检查命令

```bash
cd orin-frontend

# 安装依赖（首次）
npm install

# 开发模式
npm run dev

# 构建
npm run build

# 运行测试
npm run test

# Lint 检查
npm run lint

# Lint 自动修复
npm run lint:fix
```

### 常用检查组合

```bash
# 快速检查（lint）
npm run lint

# 构建检查
npm run build

# 测试检查
npm run test
```

---

## 三、AI 引擎 (orin-ai-engine)

### 技术栈
- Python 3.11 + FastAPI
- Poetry

### 最小检查命令

```bash
cd orin-ai-engine

# 安装依赖（首次）
poetry install

# 或激活已有虚拟环境
source venv/bin/activate

# 运行测试
python -m pytest tests/ -v

# 运行特定测试
python -m pytest tests/test_llm_handler.py -v
```

### 常用检查组合

```bash
# 快速语法检查
python -m py_compile app/**/*.py

# 运行所有测试
python -m pytest tests/ -v

# 带详细输出
python -m pytest tests/ -vv
```

---

## 四、最小检查清单（PR 合并前）

| 端 | 命令 | 说明 |
|---|------|------|
| 后端 | `mvn compile` | 编译检查 |
| 后端 | `mvn test` | 测试检查 |
| 前端 | `npm run lint` | 代码规范检查 |
| 前端 | `npm run build` | 构建检查 |
| AI引擎 | `python -m pytest tests/` | 测试检查 |

---

## 五、CI 建议

建议 PR 至少通过以下最小组合：

```yaml
# .github/workflows/ci.yml 示例
jobs:
  backend:
    steps:
      - run: mvn compile
      - run: mvn test

  frontend:
    steps:
      - run: npm run lint
      - run: npm run build

  ai-engine:
    steps:
      - run: python -m pytest tests/
```
