# 敏感操作二次确认与审计增强

> H1.4: 为管理端敏感操作补二次确认或审计增强

---

## 已有的审计能力

系统已实现审计日志模块：

| 字段 | 说明 |
|------|------|
| `userId` | 操作人 ID |
| `apiKeyId` | API Key ID |
| `endpoint` | 请求路径 |
| `method` | 请求方法 |
| `statusCode` | 响应状态码 |
| `traceId` | 全链路追踪 ID |
| `createdAt` | 操作时间 |

---

## 敏感操作清单

以下操作需要二次确认：

| 操作 | 页面 | 确认方式 | 审计字段 |
|------|------|----------|----------|
| 删除用户 | 用户管理 | 弹窗确认 + 输入用户名 | `DELETE_USER` |
| 删除知识库 | 知识库 | 弹窗确认 | `DELETE_KB` |
| 删除智能体 | 智能体 | 弹窗确认 | `DELETE_AGENT` |
| 数据恢复 | 系统维护 | 弹窗确认 + 验证码 | `DATA_RESTORE` |
| 系统升级 | 系统维护 | 弹窗确认 | `SYSTEM_UPGRADE` |
| 配置重置 | 系统设置 | 弹窗确认 | `CONFIG_RESET` |
| 权限变更 | 角色管理 | 弹窗确认 | `PERMISSION_CHANGE` |

## 前端二次确认实现

当前已使用 `ElMessageBox.confirm` 的页面：

```javascript
// 用户删除 (UserManagement.vue:452)
await ElMessageBox.confirm(
  `确定要删除用户 ${row.username} 吗？此操作不可恢复！`,
  '确认删除',
  {
    confirmButtonText: '删除',
    cancelButtonText: '取消',
    type: 'warning'
  }
)
```

## 审计增强建议

### 1. 增加操作类型字段

建议在 `AuditLog` 实体增加 `action` 字段：

```java
private String action;  // USER_DELETE, KB_DELETE, CONFIG_CHANGE 等
```

### 2. 增加敏感操作标记

```java
private boolean sensitive;  // 是否敏感操作
private String reason;      // 操作原因 (可选)
```

### 3. 增强记录内容

```java
// 当前已有
userId, endpoint, method, statusCode, traceId

// 建议增加
action, requestBody (脱敏), responseBody (脱敏), ipAddress
```
