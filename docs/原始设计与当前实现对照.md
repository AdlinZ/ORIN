# 原始设计方案 vs 当前实现对照说明

本文档简要对比原始设计文档与当前实现之间的差异，供产品和开发统一理解。

---

## 设计来源

- **原始设计**: `智能体管理系统设计方案.docx` (Yuxi-Know/FastAPI 架构)
- **当前实现**: Spring Boot + Vue 3 + Python AI Engine

---

## 架构差异

| 设计项 | 原始设计 | 当前实现 | 说明 |
|--------|---------|---------|------|
| 后端架构 | FastAPI 单体 | Spring Boot + Python Engine | 双引擎结构 |
| 前端 | 未明确 | Vue 3 + Element Plus | 保持不变 |
| 数据库 | 单一 DB | MySQL + Milvus | 向量库分离 |
| API 网关 | 内置 | 统一网关 `/v1/*` | 保持不变 |

---

## 功能模块对照

| 模块 | 原始设计 | 当前实现 | 状态 |
|------|---------|---------|------|
| 智能体管理 | Dify/RAGFlow 集成 | 多厂商接入 (Kimi/DeepSeek/MiniMax/Zhipu) | ✅ 完成 |
| 知识库 | 向量检索 | Milvus 向量库 + 多种解析器 | ✅ 完成 |
| 工作流 | DSL 执行 | Java 编排 + Python 执行器 | ✅ 完成 |
| 多智能体协作 | 任务编排 | Collaboration 包/子任务/事件 | ⚠️ 85% |
| 监控 | Prometheus | Prometheus + Langfuse + 本地 Trace | ⚠️ 需完善 |
| 端侧同步 | Webhook | 变更查询 + 导出 + Webhook | ✅ 完成 |

---

## 接口路径差异

| 功能 | 原始设计 | 当前实现 |
|------|---------|---------|
| Agent 调用 | `/v1/chat/completions` | `/api/v1/agents/{id}/chat` |
| 工作流执行 | `/v1/workflows/run` | `/api/v1/workflow/run` |
| 协作任务 | `/v1/collaboration/*` | `/api/v1/collaboration/*` |

---

## 待收敛项

1. **状态定义**: TaskEntity vs TaskQueue 状态语义需统一
2. **监控入口**: Langfuse/Trace/Statistics 三类数据需统一展示
3. **重试策略**: 不同任务类型需支持差异化配置
