# ORIN 开发任务分段书

## 文档说明

本文档将 ORIN 项目开发任务拆分为单次 Claude Code 对话可完成的粒度。每个任务应可在 1-2 小时内完成。

---

## 阶段 A：真相对齐与基础稳定化（2-3周）

### A1: 文档与状态统一

- [x] **A1.1** 更新 README.md，删除或降级未闭环能力描述
- [x] **A1.2** 为每个模块增加状态标签：`已上线`、`内测`、`开发中`、`占位`
- [x] **A1.3** 统一口径，不使用"90%+完成度"说法

### A2: 菜单与接口收敛

- [x] **A2.1** 隐藏或标记知识图谱等占位入口
- [x] **A2.2** 为未实现的控制器接口增加明确错误响应
- [x] **A2.3** 为批量导入导出、刷新元数据、异步任务状态增加禁用提示

### A3: 测试与环境修复

- [x] **A3.1** 修复后端测试配置错误（移除非法的 spring.profiles.active）
- [x] **A3.2** 明确 Python 引擎测试安装方式，补充 pytest 依赖
- [x] **A3.3** 固化前端、后端、AI 引擎的本地开发启动步骤

### A4: 配置边界收敛

- [x] **A4.1** 明确 dev、test、prod 三套环境变量来源
- [x] **A4.2** 检查数据库、Milvus、Redis、外部模型接口的必填配置
- [x] **A4.3** 给系统设置增加基础参数校验

## 阶段 B：主链路闭环（4-6周）

- [x] **B1.1** 真正实现批量导出功能
- [x] **B1.2** 真正实现批量导入功能
- [x] **B1.3** 真正实现刷新全部元数据
- [x] **B1.4** 真正实现异步任务状态查询
- [x] **B1.5** 为聊天、导入导出、刷新操作补审计记录

### B2: 工作流执行器补完

- [x] **B2.1** 修复工作流代理的 provider 注入逻辑
