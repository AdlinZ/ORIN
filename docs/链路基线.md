# ORIN 链路基线文档

本文档定义三条主链路的命名、边界、关键点和层级关系。

---

## 1. 三条主链路定义

### 1.1 Agent 调用链 (Agent Call Chain)

**链路名称**: Agent 调用链

**核心职责**: 智能体路由、模型调用、审计、Trace ID 透传

| 项目 | 内容 |
|------|------|
| 入口接口 | `/api/v1/agents/{agentId}/chat` |
| 入口接口(备) | `/v1/chat` (统一网关) |
| 执行服务 | `AgentExecutor`, `AgentManageService` |
| 审计点 | `AuditLogService` 记录用户、agent、请求参数、响应、耗时、token |
| 追踪点 | `traceId` 贯穿 `AgentManageController` → `AgentExecutor` → LLM Provider |
| 失败处理 | 异常抛至 `GlobalExceptionHandler`，记录错误日志和审计 |

### 1.2 Workflow 执行链 (Workflow Execution Chain)

**链路名称**: Workflow 执行链

**核心职责**: 工作流定义管理、实例创建、执行代理到 Python AI Engine、执行审计

| 项目 | 内容 |
|------|------|
| 入口接口 | `/api/workflows` (创建/管理) |
| 入口接口(执行) | `/api/v1/workflow/run`, `/api/workflows/{id}/execute` |
| 执行服务 | `WorkflowService`, `GraphExecutor` (Java), `PythonEngine` (FastAPI) |
| 审计点 | `AuditLogService` 记录 workflowId, instanceId, 节点执行结果 |
| 追踪点 | `traceId` 贯穿 Java → Python Engine |
| 失败处理 | 节点执行失败可重试、可跳过，异常记录至审计和事件总线 |

### 1.3 Collaboration 协作链 (Collaboration Chain)

**链路名称**: Collaboration 协作链

**核心职责**: 任务包创建、子任务分解、状态推进、事件记录、回退

| 项目 | 内容 |
|------|------|
| 入口接口 | `/api/v1/collaboration/packages` |
| 入口接口(编排) | `/api/v1/collaboration/orchestrate` |
| 执行服务 | `CollaborationOrchestrator`, `CollaborationExecutor`, `CollaborationService` |
| 审计点 | `CollabEventLogEntity` 记录包/子任务状态变更、结果 |
| 追踪点 | `traceId` + `packageId` + `subtaskId` 联合追踪 |
| 失败处理 | 子任务失败可重试、可跳过、可触发回退；包失败有 `FALLBACK` 状态 |

## 2. 链路关键点对照表

### 2.1 入口接口

| 链路 | HTTP 方法 | 路径 | 控制器 |
|------|----------|------|--------|
| Agent 调用链 | POST | `/api/v1/agents/{agentId}/chat` | `AgentManageController` |
| Agent 调用链 | POST | `/v1/chat` | `ApiGatewayController` |
| Workflow 执行链 | POST | `/api/workflows` | `WorkflowController` |
| Workflow 执行链 | POST | `/api/v1/workflow/run` | `WorkflowProxyController` |
| Collaboration 协作链 | POST | `/api/v1/collaboration/packages` | `CollaborationController` |
| Collaboration 协作链 | POST | `/api/v1/collaboration/orchestrate` | `CollaborationOrchestratorController` |

### 2.2 执行服务

| 链路 | 核心执行服务 | 调用目标 |
|------|-------------|----------|
| Agent 调用链 | `AgentExecutor` | 各 Provider (Kimi/DeepSeek/MiniMax/Zhipu/SiliconFlow) |
| Workflow 执行链 | `GraphExecutor` + `WorkflowProxyController` | Python AI Engine |
| Collaboration 协作链 | `CollaborationExecutor` | Agent / Workflow / 人工任务 |

### 2.3 审计记录

| 链路 | 审计字段 |
|------|----------|
| Agent 调用链 | userId, agentId, conversationId, request, response, duration, tokens, traceId |
| Workflow 执行链 | userId, workflowId, instanceId, nodeId, status, duration, traceId |
| Collaboration 协作链 | packageId, subtaskId, eventType, status, result, traceId |

### 2.4 追踪点 (traceId)

- 三条链路均使用统一 `traceId`，通过 MDC 或请求头传递
- Agent 调用链: API → Service → Provider
- Workflow 执行链: API → Service → Python Engine (透传)
- Collaboration 协作链: API → Orchestrator → Subtask Executor → Agent/Workflow

### 2.5 失败处理

| 链路 | 失败状态 | 处理方式 |
|------|----------|----------|
| Agent 调用链 | 500/502/504 | 记录审计 + 异常返回 |
| Workflow 执行链 | FAILED | 节点失败可重试/跳过，最终状态 FAILED |
| Collaboration 协作链 | FAILED | 子任务失败可重试/跳过/回退，包有 FALLBACK 状态 |

## 3. 任务层级关系

### 3.1 任务对象层级

```
业务任务 (TaskService / TaskEntity)
    │
    ├── 工作流任务 (WorkflowService → TaskEntity)
    │   - 关联 workflowId, instanceId
    │   - 通过 TaskQueue 队列执行
    │
    ├── 协作任务 (CollaborationService → CollaborationPackageEntity)
    │   - 包含 subtasks (CollabSubtaskEntity)
    │   - 通过 CollaborationOrchestrator 编排
    │
    └── 重试任务 (TaskQueueService → TaskQueue)
        - 指数退避重试
        - 死信队列
```

### 3.2 状态统一

| 通用状态 | TaskEntity | TaskQueue | Collaboration |
|----------|------------|-----------|---------------|
| PENDING | QUEUED | PENDING | PLANNING/PENDING |
| RUNNING | RUNNING | RUNNING | RUNNING/DECOMPOSING/EXECUTING |
| COMPLETED | COMPLETED | COMPLETED | COMPLETED/CONSENSUS |
| FAILED | FAILED | FAILED | FAILED/FALLBACK |

> 注: 当前 `TaskEntity` 和 `TaskQueue` 状态定义略有差异，待后续收敛。

## 4. 执行流程图

### 4.1 Agent 调用链

```
User Request
    │
    ▼
AgentManageController.chat()
    │
    ▼
AgentExecutor.execute()
    │
    ├──▶ AuditLogService.log() ← 审计点
    │
    └──▶ LLM Provider (Kimi/DeepSeek/...)
         │
         ▼
    Response + traceId
```

### 4.2 Workflow 执行链

```
User Request
    │
    ▼
WorkflowController.create() / execute()
    │
    ▼
WorkflowService.createInstance()
    │
    ▼
GraphExecutor.execute()
    │
    ├──▶ Python Engine (FastAPI)
    │     │
    │     ▼
    │   Nodes Execution
    │
    ├──▶ AuditLogService.log()
    │
    └──▶ Response + traceId
```

### 4.3 Collaboration 协作链

```
User Request (创建任务包)
    │
    ▼
CollaborationOrchestratorController.orchestrate()
    │
    ▼
CollaborationOrchestrator.decompose()
    │
    ├──▶ CollaborationService.createSubtasks()
    │
    ▼
CollaborationExecutor.executeSubtasks()
    │
    ├──▶ For each subtask:
    │     ├──▶ AgentExecutor (Agent调用链)
    │     ├──▶ GraphExecutor (Workflow执行链)
    │     └──▶ Manual Task (人工处理)
    │
    ├──▶ CollabEventLogEntity.log()
    │
    ▼
CollaborationOrchestrator.checkCompletion()
```
