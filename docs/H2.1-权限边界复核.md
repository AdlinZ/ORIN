# H2.1: 权限边界复核

## 一、权限模型概述

### 实体结构

| 实体 | 说明 | 关键字段 |
|------|------|----------|
| `SysUser` | 用户 | `userId`, `username`, `password`, `role`(String), `departmentId` |
| `SysRole` | 角色 | `roleId`, `roleCode`(ROLE_ADMIN/ROLE_USER), `departmentId` |
| `SysDepartment` | 部门 | `departmentId`, `departmentName`, `parentId`(层级) |
| `ApiKey` | API密钥 | `keyHash`, `userId`, `permissions`, `rateLimit*`, `expiresAt` |

### 认证机制

1. **JWT Token** - 用于管理端用户认证（Header: `Authorization: Bearer eyJ...`）
2. **API Key** - 用于外部调用认证（Header: `X-API-Key: sk-orin-xxx`）

## 二、严重安全发现

### 问题：excludePathPatterns 导致权限绕过

**位置**: `WebConfig.java` (line 62-80)

```java
registry.addInterceptor(apiKeyAuthInterceptor)
    .addPathPatterns("/v1/**", "/api/v1/**")
    .excludePathPatterns(
        "/api/v1/agents/**",       // 智能体管理 - 无需认证
        "/api/v1/model-config/**", // 模型配置 - 无需认证
        "/api/v1/departments/**",   // 部门管理 - 无需认证
        "/api/v1/roles/**",        // 角色管理 - 无需认证
        "/api/v1/users/**"         // 用户管理 - 无需认证
    );
```

**风险**：这些端点被排除在 API Key 验证之外，如果请求也没有 JWT Token，则完全没有任何认证。

---

## 三、SecurityConfig vs Interceptor 关系

| 配置层 | 作用 | 状态 |
|--------|------|------|
| `SecurityConfig` | URL级权限配置 | `permitAll()` |
| `ApiKeyAuthInterceptor` | API Key验证 | 部分路径被exclude |
| `@PreAuthorize` | 方法级权限校验 | 依赖Authentication对象 |

**问题链**：
1. `SecurityConfig` 配置 `/api/v1/agents/**` 为 `permitAll()`
2. `ApiKeyAuthInterceptor` 跳过这些路径（excludePathPatterns）
3. 如果请求无 JWT → 无 Authentication 对象
4. `@PreAuthorize("hasRole('ADMIN')")` 无法获取角色信息 → 可能拒绝或静默放行

## 四、ApiKeyController 权限检查

```java
@PreAuthorize("hasRole('ADMIN')")
@PostMapping      // 创建 API Key
@PreAuthorize("hasRole('ADMIN')")
@DeleteMapping("/{id}")  // 删除 API Key
@PreAuthorize("hasRole('ADMIN')")
```

**分析**：所有管理操作都有 `@PreAuthorize("hasRole('ADMIN')")`

**但问题**：如果请求没有经过任何认证（无JWT、无API Key），`Authentication` 为 null，`@PreAuthorize` 如何处理？

## 六、建议修复

### 1. 移除不必要的 excludePathPatterns

如果这些端点需要 JWT 认证，不应该在 interceptor 层面 exclude：

```java
.excludePathPatterns(
    // 移除这些，或确保它们通过JWT认证
    "/api/v1/agents/**",       // 应该需要JWT
    "/api/v1/departments/**",   // 应该需要JWT
    "/api/v1/roles/**",        // 应该需要JWT
    "/api/v1/users/**"         // 应该需要JWT
)
```

### 2. 确认 @PreAuthorize 能正常工作

确保 excludePathPatterns 的端点：
- 要么有有效的 JWT Token（JwtAuthenticationFilter 设置 Authentication）
- 要么有有效的 API Key（但这些端点被 exclude 了）

### 3. 统一权限模型

建议 SysUser.role 从 String 改为关联 SysUserRole 表，实现标准的 Many-to-Many 角色模型。
