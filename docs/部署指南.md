# ORIN 部署指南

本文档只保留当前仓库里还能对应到脚本和代码的部署方式，去掉已经和实际目录、端口、配置方式不一致的旧内容。

## 组件结构

ORIN 运行时通常包含三部分：

- `orin-backend`：Spring Boot，默认 `8080`
- `orin-frontend`：Vite 开发服务器，默认 `5173`
- `orin-ai-engine`：FastAPI，默认 `8000`

常见依赖：

- Java 17+
- Node.js 18+ 更稳妥
- Python 3.10+
- MySQL 8.0+
- Redis 6.0+（推荐）

可选依赖：

- FFmpeg：音视频处理
- Tesseract OCR：本地 OCR

## 本地开发部署

### 1. 准备环境

```bash
cd /path/to/ORIN
```

后端环境变量建议先按 [ENVIRONMENT_SETUP.md](../ENVIRONMENT_SETUP.md) 配置。

最少需要确认：

- 数据库可连通
- `JWT_SECRET` 已设置
- 后端 `.env` 或系统环境变量中有数据库账号密码

### 2. 创建数据库

```sql
CREATE DATABASE orindb CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'orin_user'@'localhost' IDENTIFIED BY 'your_secure_password';
GRANT ALL PRIVILEGES ON orindb.* TO 'orin_user'@'localhost';
FLUSH PRIVILEGES;
```

### 3. 后端配置

推荐方式：

```bash
cd orin-backend
cp .env.example .env
```

然后按实际环境填写数据库和 JWT 配置。

### 4. 启动服务

#### 方式 A：使用根目录脚本

```bash
./manage.sh start
```

脚本行为说明：

- 会尝试检查并启动 MySQL
- 会编译后端并启动 `8080`
- 会启动 AI 引擎到 `8000`
- 会启动前端开发服务器到 `5173`

如果你不是 macOS + 本地 MySQL 这类环境，脚本可能并不完全适配，此时建议使用手动启动。

#### 方式 B：手动启动

后端：

```bash
cd orin-backend
mvn spring-boot:run
```

AI 引擎：

```bash
cd orin-ai-engine
venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000
```

如果没有 `venv`，需要先自行准备 Python 虚拟环境并安装依赖。

前端：

```bash
cd orin-frontend
npm install
npm run dev
```

### 5. 验证

- 前端：`http://localhost:5173`
- 后端：`http://localhost:8080`
- Swagger UI：`http://localhost:8080/swagger-ui/index.html`
- OpenAPI：`http://localhost:8080/v3/api-docs`
- 健康检查：`http://localhost:8080/api/v1/health`

补充提醒：

- 能访问页面不代表前后端已全部对接完成
- 能打开 Swagger 不代表对应页面已经接上这些接口
- 部署验证应优先做最小链路，而不是只看端口和首页

## 生产部署建议

生产环境建议不要直接照搬 `manage.sh`，而是拆成独立服务。

### 1. 构建

后端：

```bash
cd orin-backend
mvn clean package -DskipTests
```

前端：

```bash
cd orin-frontend
npm install
npm run build
```

AI 引擎：

```bash
cd orin-ai-engine
python -m venv venv
venv/bin/pip install -e .
```

当前仓库使用 `pyproject.toml`，不应再假设存在长期维护的 `requirements.txt`。

### 2. 环境变量

生产环境至少应显式配置：

- `SPRING_PROFILES_ACTIVE=prod`
- `DB_HOST`
- `DB_PORT`
- `DB_NAME`
- `DB_USER`
- `DB_PASSWORD`
- `JWT_SECRET`

如果用到以下功能，还需要补充：

- 邮件通知：`MAIL_*`
- 告警机器人：`DINGTALK_*`、`WECHAT_*`
- SiliconFlow / 多模态能力相关 key
- 向量库、RAGFlow、Dify 等外部集成配置

### 3. Systemd 示例

后端：

```ini
[Unit]
Description=ORIN Backend
After=network.target mysql.service

[Service]
Type=simple
User=orin
WorkingDirectory=/opt/orin/orin-backend
EnvironmentFile=/opt/orin/.env
ExecStart=/usr/bin/java -jar /opt/orin/orin-backend/target/orin-backend-1.0.0-SNAPSHOT.jar
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

AI 引擎：

```ini
[Unit]
Description=ORIN AI Engine
After=network.target

[Service]
Type=simple
User=orin
WorkingDirectory=/opt/orin/orin-ai-engine
ExecStart=/opt/orin/orin-ai-engine/venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

### 4. Nginx 反向代理示例

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        root /opt/orin/orin-frontend/dist;
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /v1/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## 常见坑位

### 1. `manage.sh` 里的数据库假设较强

- 默认数据库名是 `orindb`
- 默认用户是 `root`
- 默认密码是 `password`
- 可以通过 `ORIN_DB_NAME`、`ORIN_DB_USER`、`ORIN_DB_PASS` 覆盖

### 2. AI 引擎端口不是 8088

当前根目录脚本默认启动在 `8000`，旧文档里如果提到 `8088`，以当前脚本和部署配置为准。

### 3. Swagger 路径优先使用新地址

优先使用：

- `http://localhost:8080/swagger-ui/index.html`

而不是继续写旧的 `swagger-ui.html`。

### 4. 多模态相关功能依赖外部组件

- 图片 OCR 可能需要 Tesseract 或云端 OCR
- 音频转写可能依赖 FFmpeg 和 ASR 服务
- 没配好时接口不一定报错失败，但结果可能是降级文本
