# 版本发布流程

> I2.1: 约定版本号、变更记录、数据库迁移发布顺序

---

## 版本号规则

采用 `主版本.次版本.修订号` 格式：

- **主版本 (MAJOR)**: 重大架构变更，不兼容 API
- **次版本 (MINOR)**: 新增功能，向后兼容
- **修订号 (PATCH)**: Bug 修复，向后兼容

### 当前版本

```
v1.0.0 (首个稳定版本)
```

---

## 发布流程

### 1. 准备阶段

- [ ] 从 `main` 创建发布分支 `release/vX.Y.Z`
- [ ] 更新版本号
- [ ] 生成变更日志 (Changelog)
- [ ] 执行完整测试

### 2. 数据库迁移

```
1. 生成 SQL 迁移脚本 (Flyway)
2. 在测试环境验证
3. 生产部署前备份
4. 执行迁移
5. 验证迁移结果
```

### 3. 部署阶段

- [ ] 后端 Docker 镜像构建
- [ ] 前端生产构建
- [ ] 灰度发布 (1% → 10% → 100%)
- [ ] 验证核心功能

### 4. 收尾阶段

- [ ] 合并到 main 和 develop
- [ ] 打标签 `git tag vX.Y.Z`
- [ ] 更新发布说明
