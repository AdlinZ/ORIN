# 数据备份恢复策略

> H3.3: 整理数据库、上传文件、图谱数据的备份恢复策略

---

## 备份内容

| 类型 | 备份方式 | 频率 | 保留期 |
|------|----------|------|--------|
| 数据库 | mysqldump / pg_dump | 每日 | 30天 |
| 配置文件 | 文件复制 | 每次变更 | 10个版本 |
| 上传文件 | rsync/tar | 每周 | 7天 |
| 图谱数据 | 导出 JSON | 每日 | 14天 |

---

## 备份脚本示例

```bash
#!/bin/bash
# 每日数据库备份
DATE=$(date +%Y%m%d)
mysqldump -u orin -p orin_db > /backup/orin_db_$DATE.sql
gzip /backup/orin_db_$DATE.sql

# 清理 30 天前的备份
find /backup -name "*.sql.gz" -mtime +30 -delete
```

---

## 恢复步骤

### 数据库恢复

```bash
# 全量恢复
gunzip < orin_db_20260328.sql.gz | mysql -u orin -p orin_db

# 单表恢复
mysql -u orin -p orin_db -e "source restore_table.sql"
```

### 文件恢复

```bash
# 从备份 tar 包恢复
tar -xzf uploads_backup_20260328.tar.gz -C /data/uploads/
```
