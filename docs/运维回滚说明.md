# 运维操作回滚说明

> H3.2: 给高风险运维操作补回滚说明

---

## 高风险操作清单

| 操作 | 风险等级 | 回滚方式 | 恢复时间 |
|------|----------|----------|----------|
| 数据备份 | 🟢 低 | 无需回滚 | - |
| 数据恢复 | 🔴 高 | 重新备份覆盖 | 视数据量 |
| 系统升级 | 🟡 中 | 降级版本 | 10-30min |
| 日志归档 | 🟢 低 | 无需回滚 | - |
| 缓存清理 | 🟢 低 | 无需回滚 | <1min |

---

## 回滚步骤

### 数据恢复回滚

```bash
# 1. 停止服务
systemctl stop orin-backend

# 2. 从最新备份恢复
mysql -u orin -p orin_db < /backup/orin-2026-03-28.sql

# 3. 重启服务
systemctl start orin-backend
```

### 系统升级回滚

```bash
# 1. 停止服务
docker-compose down

# 2. 回滚镜像版本
docker-compose.yml 中修改:
  image: orin-backend:上一版本标签

# 3. 重启
docker-compose up -d
```
