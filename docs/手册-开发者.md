# ORIN 开发者手册

本文档面向 ORIN 平台的开发者，涵盖本地调试、API 接入、MCP 集成与代码规范。

---

## 1. 本地开发环境

### 1.1 快速启动（进程模式）

```bash
# 前置依赖：JDK 17+、Node 18+、Python 3.11+、MySQL 8、Redis
# 启动全部服务（后端 + 前端 + AI Engine）
./manage.sh start

# 查看服务状态
./manage.sh status

# 停止全部服务
./manage.sh stop

# 重启某个服务
./manage.sh restart backend
```

### 1.2 Docker 模式开发

```bash
# 仅启动基础设施（MySQL / Redis / RabbitMQ），后端和前端用进程模式
docker compose up -d mysql redis rabbitmq

# 验证基础设施
bash scripts/check-docker-quickstart.sh

# 分别启动后端和前端
cd orin-backend && mvn spring-boot:run
cd orin-frontend && npm run dev
```

### 1.3 环境变量

关键环境变量（`.env`）：

```bash
# 数据库（必须）
MYSQL_HOST=localhost
MYSQL_PORT=3306
MYSQL_ROOT_PASSWORD=your_password
REDIS_HOST=localhost
REDIS_PASSWORD=

# AI Engine（必须）
ORIN_AI_ENGINE_URL=http://localhost:8000

# 后端（必须）
ORIN_DEFAULT_ADMIN_PASSWORD=your_admin_password
JWT_SECRET=your_jwt_secret_min_32_chars

# 可选：启用 OTel 链路追踪
ORIN_TRACING_ENABLED=true
OTEL_EXPORTER_OTLP_ENDPOINT=http://localhost:4317

# 可选：Langfuse LLM 观测
LANGFUSE_ENABLED=true
LANGFUSE_PUBLIC_KEY=
LANGFUSE_SECRET_KEY=
LANGFUSE_HOST=http://localhost:3000
```

## 2. API 接入

### 2.1 获取 API Key

1. 登录管理台 → 系统设置 → API Key 管理
2. 创建新密钥，复制 `sk-orin-...` 前缀的密钥
3. 密钥仅显示一次，务必保存

### 2.2 API 调用示例

```bash
# 设置变量
API_KEY="sk-orin-your-key-here"
BASE_URL="http://localhost:8080"

# 1. 登录获取 JWT（或者使用 API Key 直接访问）
curl -X POST "${BASE_URL}/api/v1/auth/login" \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"your_password"}'

# 2. Agent 列表（API Key 鉴权）
curl "${BASE_URL}/api/v1/agents" \
  -H "Authorization: Bearer ${API_KEY}"

# 3. 创建 Workflow
curl -X POST "${BASE_URL}/api/v1/workflow/publish" \
  -H "Authorization: Bearer ${API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{"name":"test","dsl":{...}}'

# 4. 执行 Workflow
curl -X POST "${BASE_URL}/api/v1/workflow/run" \
  -H "Authorization: Bearer ${API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{"workflowId":"..."}'

# 5. MCP 端点（Streamable HTTP）
curl -X POST "${BASE_URL}/v1/mcp" \
  -H "Authorization: Bearer ${API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","method":"tools/list","id":1}'
```

### 2.3 SDK 使用（Python）

```python
import requests

ORIN_BASE_URL = "http://localhost:8080"
API_KEY = "sk-orin-your-key"

headers = {
    "Authorization": f"Bearer {API_KEY}",
    "Content-Type": "application/json"
}

# 获取 Agent 列表
agents = requests.get(f"{ORIN_BASE_URL}/api/v1/agents", headers=headers)
print(agents.json())

# 执行 Workflow
result = requests.post(
    f"{ORIN_BASE_URL}/api/v1/workflow/run",
    headers=headers,
    json={"workflowId": "wf-xxx", "input": {"key": "value"}}
)
print(result.json())
```

## 3. MCP 集成

### 3.1 MCP 客户端配置

ORIN 作为 MCP Server，支持 Codex / Claude Desktop / Cursor / Windsurf 等客户端。

**Codex 配置（`~/.config/orin/config.json`）**：
```json
{
  "mcpServers": {
    "orin": {
      "url": "http://localhost:8080/v1/mcp",
      "headers": {
        "Authorization": "Bearer sk-orin-your-key"
      }
    }
  }
}
```

详细配置见 [docs/mcp-client-setup.md](./mcp-client-setup.md)。

### 3.2 MCP 工具列表

```bash
# 查看可用工具
curl -X POST "http://localhost:8080/v1/mcp" \
  -H "Authorization: Bearer sk-orin-your-key" \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","method":"tools/list","id":1}' | jq '.result.tools[].name'
```

可用工具：
- `agent.chat` — 与智能体对话
- `agent.list` — 列出所有 Agent
- `workflow.run` — 执行工作流
- `workflow.list` — 列出所有工作流
- `workflow.publish` — 发布新工作流

### 3.3 在代码中使用 MCP

```python
# orin-mcp-bridge 用法
from orin_mcp_bridge import load_config, create_server

config = load_config()  # 读取 ORIN_BASE_URL, ORIN_API_KEY
app = create_server(config)

# 桥接后 ORIN 作为 MCP Server 暴露所有工具
# 任何 MCP 客户端都可以通过 stdio 连接使用
```

## 4. 本地调试

### 4.1 后端调试

```bash
# 方式一：IDE 启动（推荐）
# 在 IDE 中直接运行 OrinBackendApplication.java，设置断点

# 方式二：Maven 启动
cd orin-backend
mvn spring-boot:run -Dspring-boot.run.arguments="--debug"

# 方式三：附加调试端口
java -agentlib:jdwp=transport=dt_socket,server=y,suspend=n,address=5005 \
     -jar target/orin-backend-1.0.0.jar
```

### 4.2 前端调试

```bash
cd orin-frontend

# 开发模式（热重载）
npm run dev

# 查看 Vue 组件 props / state
# 在浏览器 Vue DevTools 中打开

# API 请求抓包
# 浏览器 DevTools → Network → 过滤 /api/v1
```

### 4.3 AI Engine 调试

```bash
cd orin-ai-engine

# 激活虚拟环境
source .venv/bin/activate

# 启动（开发模式，reload=True）
uvicorn app.main:app --reload --port 8000 --log-level debug

# 查看 LangGraph 执行日志
# AI Engine 会输出每个节点的输入输出
```

### 4.4 Trace 调试

```bash
# 搜索特定 traceId 的日志
grep "traceId.*abc123def456" orin-backend/logs/orin-backend.json | jq .

# 查看 JSON 结构日志字段
head -1 orin-backend/logs/orin-backend.json | jq 'keys'

# 常见 trace 相关 header
# traceparent: 00-<traceId>-<spanId>-01  (W3C 标准)
# X-Trace-Id: <traceId>                   (ORIN 内部)
```

## 6. 代码规范

### 6.1 提交规范

```
<type>(<scope>): <subject>

Types: feat / fix / refactor / docs / test / chore
Scope: backend / frontend / ai-engine / docker / scripts / docs

示例：
feat(collaboration): add FALLBACK retry mechanism
fix(workflow): correct loop node output writing
docs(api): update API documentation for trace endpoint
```

### 6.2 分支规范

```
main          — 生产可发布代码
├── develop   — 开发主分支（如果使用 develop 分支）
├── feat/*    — 新功能
├── fix/*     — 修复
└── docs/*    — 文档
```

### 6.3 API 规范

- 所有 API 端点必须携带 `Authorization: Bearer <token/key>`
- 响应格式统一：`Result<T>`（包含 `code` / `message` / `data` / `traceId`）
- 错误响应必须包含 `traceId` 供排查
- 新接口必须加 Swagger `@Operation` 注解

详细规范见 [docs/开发规范.md](./开发规范.md)。

## 7. 常见问题

### Q: 后端启动报 `Database 'orindb' does not exist`

```bash
# 初始化数据库
docker compose up -d mysql
# Flyway 会自动执行 schema 初始化
# 如果是干净环境，确认 docker/mysql/init/01-orin-schema.sql 存在
```

### Q: AI Engine 调用返回 500

```bash
# 1. 检查 AI Engine 是否在线
curl http://localhost:8000/health

# 2. 检查 ORIN_AI_ENGINE_URL 配置
grep "ORIN_AI_ENGINE_URL" .env

# 3. 检查 AI Engine 日志
docker compose logs ai-engine --tail=30
```

### Q: MCP 工具调用返回 401

```bash
# 确认 API Key 正确且未过期
# 在管理台检查 Key 状态（启用/禁用）
# 确认 Authorization header 格式正确：Bearer sk-orin-xxx
```

### Q: traceparent 在 AI Engine 中看不到

```bash
# 确认 ORIN 后端已注入 traceparent
# 在 AI Engine 日志中搜索 traceparent
docker compose logs ai-engine | grep traceparent

# 或检查请求 header
curl -v http://localhost:8000/api/v1/run \
  -H "traceparent: 00-abcdef123456-1234567890abcdef-01"
```
