# 超时/超大文件/超配额统一错误提示

> G2.3: 给超时、超大文件、超配额场景补统一错误提示

---

## 错误码定义

在 `ErrorCode.java` 中已有以下相关错误码：

| 错误码 | 说明 | 提示文案 |
|--------|------|----------|
| `10001` | 参数验证失败 | 参数格式不正确，请检查输入 |
| `10002` | 操作失败 | 操作失败，请稍后重试 |
| `10005` | 请求过于频繁 | 请求过于频繁，请稍后重试 |
| `20003` | 资源冲突 | 资源冲突，请稍后重试 |
| `20004` | 资源已锁定 | 资源已被锁定，请稍后重试 |

## 场景与提示

### 超时场景

| 场景 | 错误码 | 提示文案 |
|------|--------|----------|
| 请求超时 | `10005` | "请求超时，请稍后重试" |
| 任务执行超时 | `10002` | "任务执行超时，请检查任务状态" |
| 数据库超时 | `10000` | "数据库响应超时，请稍后重试" |
| AI 调用超时 | `50003` | "AI 服务响应超时，请稍后重试" |

### 超大文件场景

| 场景 | 错误码 | 提示文案 |
|------|--------|----------|
| 文件大小超限 | `10001` | "文件大小超过限制(10MB)，请压缩后重试" |
| 请求体过大 | `10001` | "请求数据过大，请分批处理" |
| 内存不足 | `10000` | "系统资源不足，请使用较小文件" |

### 超配额场景

| 场景 | 错误码 | 提示文案 |
|------|--------|----------|
| 请求频率超限 | `10005` | "请求频率超限，请稍后重试" |
| Token 超限 | `10002` | "Token 配额已用完，请联系管理员" |
| 存储配额超限 | `10002` | "存储配额已用完，请清理后重试" |
| 并发数超限 | `10005` | "并发数超限，请稍后重试" |

## 前端统一处理

建议在 `src/utils/message.js` 中统一处理：

```javascript
export const handleError = (error) => {
  const code = error?.code || error?.response?.status
  
  const errorMessages = {
    '10005': '请求过于频繁，请稍后重试',
    '20003': '资源冲突，请稍后重试',
    '20004': '资源已被锁定，请稍后重试',
    '50003': 'AI 服务响应超时，请稍后重试'
  }
  
  ElMessage.error(errorMessages[code] || '操作失败，请稍后重试')
}
```
