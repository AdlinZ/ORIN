# ORIN 功能完成度

本文档替代旧的"90%+ 完成度"宣传口径，按当前仓库可见代码与页面给出保守评估，并列出本地可验收能力清单。

## 1. 总体口径

- 平台骨架：约 **75–80%**（多模块入口、数据结构齐备）
- 真实业务闭环：约 **50–60%**（智能体可演示，部分高级链路待补）
- 对外稳定交付：约 **45–55%**（仍需测试体系、稳定性与一致性增强）

判断"能力是否真可用"的统一口径——同时核对：

1. 前端路由与页面是否真实挂载
2. Java 控制器与 service 是否真实闭环
3. Python AI 引擎是否参与并完成真实执行

下列情况均不计为完整交付：仅有页面入口而无后端闭环；控制器存在但 service 占位；Java 入口存在但 Python 端仍是 mock；依赖外部服务且默认配置无法运行。

- `<4.0` 仅展示或局部能力

## 4. 本地可验收能力

### 4.1 已完成（浏览器即验）

| 能力 | 路径 / 命令 | 通过口径 |
|------|-------------|----------|
| 前端入口 | `/`、`/portal` | 页面打开，无白屏 |
| 后端健康检查 | `GET /v1/health`、`GET /api/v1/health` | 无需 API Key，返回服务状态 |
| AI Engine 健康 | `GET :8000/health`、`GET :8000/v1/health` | 返回 `status` / `service` / `dependencies` |
| 旧路由兼容 | 见 `routes.js` 的 `LEGACY_ROUTE_REDIRECTS` | 历史链接可跳转到当前入口 |
| 鉴权提示 | `/dashboard/system/security-ops` | 统一网关入口明确提示 API Key 要求 |
| 工作流首屏 | 侧边栏“智能体管理 → 工作流中心” | 页面打开后工作流入口无白屏 |

### 4.2 可联调（需依赖）

| 能力 | 路径 | 依赖 |
|------|------|------|
| 知识库检索 | 侧边栏“知识库管理 → 知识检索” | MySQL · Milvus · Embedding Provider · 至少一个已入库文档 |
| 工作流执行 | 侧边栏“智能体管理 → 工作流中心” | 后端工作流 + AI Engine |
| 智能体对话 + Trace | `/dashboard/agents/manage` → 控制台 | 可用模型 provider 与 API Key |
| 统一网关统计 | `/dashboard/system/security-ops` | 真实流量 + Redis |
| MCP 连接测试 | `/dashboard/system/platform` | 本机 STDIO 命令或可达 SSE 地址 |

### 4.3 预留中（页面存在但能力未闭环）

| 能力 | 当前口径 |
|------|----------|
| MCP 工具市场完整工具列表 | 模板已展示，真实工具列表依赖具体服务接入 |
| 外部 provider 全量可用性 | 不默认可用，缺 key/模型时按统一错误语义返回 |
| RabbitMQ 协作队列 | 本地默认不强制启动，健康检查暴露 `rabbitmq` 依赖状态 |
| 知识图谱完整链路 | 实体已落地，端到端构建/检索/展示协同待完善 |
| 协作人工干预 | 后端接口已实装，前端按钮闭环未完成 |
