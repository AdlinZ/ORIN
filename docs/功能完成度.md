# ORIN 功能完成度

本文档替代旧的"90%+ 完成度"宣传口径，按当前仓库可见代码与页面给出保守评估，并列出本地可验收能力清单。

## 1. 总体口径

- 平台骨架：约 **75–80%**（多模块入口、数据结构齐备）
- 真实业务闭环：约 **55–65%**（Workflow / Collaboration / MCP 已有 API 级 smoke，真实浏览器与 provider 场景仍待补）
- 对外稳定交付：约 **50–60%**（CI、Docker quickstart、业务 smoke 与安全基线已建立，覆盖率红线、E2E、release 仍待补）

判断"能力是否真可用"的统一口径——同时核对：

1. 前端路由与页面是否真实挂载
2. Java 控制器与 service 是否真实闭环
3. Python AI 引擎是否参与并完成真实执行

下列情况均不计为完整交付：仅有页面入口而无后端闭环；控制器存在但 service 占位；Java 入口存在但 Python 端仍是 mock；依赖外部服务且默认配置无法运行。

`business-smoke.sh` 覆盖登录、临时 API Key 创建、`/v1/mcp` 鉴权成功、禁用后 401、Agent 列表、Workflow 最小 DSL 创建/发布/提交/查询、Workflow trace summary 聚合断言、已完成任务恢复保护、失败 Workflow replay、Collaboration 创建/分解/查询、Collaboration trace summary 聚合断言。Agent chat 默认 `SKIP`，需 `ORIN_BUSINESS_SMOKE_AGENT_ID=<agentId>` 才纳入必检；纳入后会要求响应不是错误负载，并在 trace summary 中查到审计或 trace 记录。协作 Workflow 子任务默认 `SKIP`，需 `ORIN_BUSINESS_SMOKE_WORKFLOW_SUBTASK=1` 才纳入必检；纳入后要求 AI Engine MQ worker、后端协作结果监听与 `ORIN_BACKEND_AUTHORIZATION` 已在服务启动时配置。`ORIN_BUSINESS_SMOKE_REQUIRE_WORKFLOW_COMPLETED` 默认为 `auto`，即 RabbitMQ 可达时要求 Workflow task 到 `COMPLETED`，也可显式设为 `true` / `false` 覆盖。
