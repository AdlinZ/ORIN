# F5.1 AgentWorkspace 知识库文件过滤检查手册

## 1. 目标
定位 `/dashboard/agents/workspace` 中“按文件过滤知识库检索”为什么未生效，并快速判断断点在前端、后端还是数据层。

## 2. 当前代码现状（截至本次检查）

1. 前端发送消息仅包含 `message` 和 `kbIds`，未包含文件过滤参数。
2. 后端 `ChatMessageRequest` 仅定义 `message`、`kbIds`。
3. 后端检索入口 `RetrievalService.hybridSearch(...)` 未接收 `docIds/documentIds` 过滤条件。
4. 会话实体 `attachedKbIds` 为 `@Transient`，数据库表 `agent_chat_session` 无持久化字段，刷新后会话侧配置可能丢失。

对应代码位置：

- `orin-frontend/src/views/Agent/AgentWorkspace.vue`（`sendMessage`）
- `orin-backend/src/main/java/com/adlin/orin/modules/conversation/dto/ChatMessageRequest.java`
- `orin-backend/src/main/java/com/adlin/orin/modules/conversation/service/AgentChatService.java`
- `orin-backend/src/main/java/com/adlin/orin/modules/knowledge/service/RetrievalService.java`
- `orin-backend/src/main/java/com/adlin/orin/modules/conversation/entity/AgentChatSession.java`
- `orin-backend/src/main/resources/db/migration/V52__Agent_Chat_Session.sql`

## 3. 检查步骤（按顺序执行）

### 3.1 前端请求体检查

1. 打开页面：`http://localhost:5173/dashboard/agents/workspace`
2. 选择知识库与文件后发送消息。
3. 打开浏览器 DevTools -> Network -> `POST /api/v1/agents/chat/sessions/{sessionId}/messages`
4. 检查 Request Payload 是否包含下列字段：
   - `message`
   - `kbIds`
   - `kbDocFilters`（或同义字段，如 `documentIdsByKb`）

判定：

- 若只有 `message/kbIds`，前端未实现文件过滤参数透传。

### 3.2 后端入参检查

1. 在 `AgentChatController.sendMessage` 或 `AgentChatService.sendMessage` 加临时日志。
2. 发送同一条消息。
3. 观察日志是否打印文件过滤字段。

判定：

- 若 Controller/Service 未接收到过滤字段，后端 DTO 未升级或反序列化未生效。

### 3.3 检索链路检查

1. 在 `RetrievalService.hybridSearch` 打印当前生效的 docId 过滤列表。
2. 在关键词检索与向量检索结果合并前后分别打印结果数量。
3. 用“明确只在某单一文档中存在”的关键词测试。

判定：

- 若日志中无 docId 过滤，说明检索层还未接入过滤。
- 若有过滤但结果仍跨文档，需检查过滤应用位置（向量路径/关键词路径是否都过滤）。

### 3.4 会话刷新一致性检查

1. 在同一会话中选择知识库和文件过滤。
2. 刷新页面并重新进入该会话。
3. 观察右侧配置是否保留。

判定：

- 若丢失，说明会话筛选状态未持久化（当前代码高概率如此）。

## 4. 快速结论模板（可复制到 issue）

```text
[结论]
- 文件过滤参数前端未透传 / 后端未接收 / 检索层未应用（按实际填写）
- 会话筛选状态刷新后丢失（是/否）

[证据]
- Network payload: ...
- Backend log: ...
- Retrieval log: ...

[影响]
- workspace 无法实现“按文件过滤查询”，结果存在跨文档污染风险
```

## 5. 最小回归用例

1. 无文件过滤，仅选知识库：应返回该知识库内多文档检索结果。
2. 单知识库 + 单文档过滤：应仅返回该文档相关 chunk。
3. 单知识库 + 多文档过滤：应仅返回所选文档并集。
4. 多知识库 + 各自文件过滤：应仅返回每个知识库各自过滤文档。
5. 刷新页面后重进会话：筛选配置应保持（若设计要求持久化）。
