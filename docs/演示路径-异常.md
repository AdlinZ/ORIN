# 异常演示路径

> I3.2: 准备 3 条异常演示路径：配置缺失、外部服务失败、权限不足

---

## 演示 1：配置缺失

### 场景
未配置 API Key 时尝试调用 AI 模型

### 步骤

1. **不配置 Key**
   - 启动服务
   - 不设置任何 Provider API Key

2. **尝试调用**
   - 创建智能体
   - 发起聊天请求

### 预期结果
- 返回错误提示："API Key 未配置"
- 不阻塞其他功能
- 日志记录错误

---

## 演示 2：外部服务失败

### 场景
外部 Provider 服务不可用时的降级行为

### 步骤

1. **模拟失败**
   - 配置错误的 API Endpoint
   - 或使用无效的 API Key

2. **触发调用**
   - 发起聊天请求

3. **观察降级**
   - 返回错误提示
   - 记录错误日志
   - 服务不崩溃

### 预期结果
- 优雅失败，提示清晰
- 熔断器触发
- 自动重试 (如配置)

---

## 演示 3：权限不足

### 场景
普通用户尝试访问管理员功能

### 步骤

1. **创建普通用户**
   - 以管理员身份创建用户
   - 分配"普通用户"角色

2. **尝试访问**
   - 使用普通用户登录
   - 尝试访问「系统管理」→「用户管理」

3. **观察拦截**
   - 返回 403 错误
   - 前端显示"无权限访问"

### 预期结果
- 后端返回 403
- 前端正确显示无权限提示
- 不泄露敏感信息

---

## 统一错误提示

| 场景 | 错误码 | 提示 |
|------|--------|------|
| 未配置 Key | 500 | "AI Provider 未配置，请联系管理员" |
| Provider 失败 | 503 | "AI 服务暂时不可用，请稍后重试" |
| 权限不足 | 403 | "无权限访问此资源" |
| 未登录 | 401 | "请先登录" |
