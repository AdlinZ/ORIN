# E2.4: 知识库模块能力边界与前置条件

## 文档说明

本文档描述 ORIN 知识库模块的能力范围、已知限制和前置依赖，作为功能验收和能力边界的参考文档。

---

## 一、模块架构概览

```
知识库模块
├── 知识库管理 (KnowledgeManageController)
│   ├── 知识库 CRUD
│   ├── 外部数据同步 (RAGFlow, Dify, Notion, Web, Database)
│   └── 智能体绑定
├── 文档管理 (DocumentManageController)
│   ├── 文件上传
│   ├── 多模态解析 (PDF/DOCX/图片/音频/视频)
│   └── 向量化与分块
├── 向量检索 (RetrievalService)
│   ├── 混合检索 (向量 + 关键词)
│   ├── Parent-Child 分层检索
│   ├── Rerank 重排序
│   └── 多模态检索
└── 知识图谱 (KnowledgeGraphController)
    ├── 图谱 CRUD
    ├── 实体管理
    └── 关系管理
```

## 二、核心能力

### 2.1 知识库管理

| 能力 | 说明 | 状态 |
|------|------|------|
| 创建知识库 | 支持 DOCUMENT / STRUCTURED / API 三种类型 | ✅ 可用 |
| 更新知识库 | 更新基础信息和配置 | ✅ 可用 |
| 删除知识库 | 级联删除文档、分块、向量记录 | ✅ 可用 |
| 知识库列表 | 获取所有知识库及绑定状态 | ✅ 可用 |
| 智能体绑定 | 智能体与知识库的多类型绑定 | ✅ 可用 |
| 知识库同步 | 从 Dify 同步最新知识库 | ✅ 可用 |

### 2.2 文档管理

#### 支持的文件类型

| 类型 | 扩展名 | 解析方式 | 向量化 |
|------|--------|----------|--------|
| 纯文本 | txt, md, markdown | 直接读取 | ✅ |
| PDF | pdf | PDFBox 文本提取 / 富文本解析 | ✅ |
| Word | docx, doc | Apache POI 提取 | ✅ |
| 图片 | png, jpg, jpeg, gif, bmp, webp | VLM OCR | ✅ |
| 音频 | mp3, wav, m4a, aac, ogg, flac | SiliconFlow ASR | ✅ |
| 视频 | mp4, mov, avi, mkv, webm, flv | FFmpeg 提取音频 → ASR | ✅ |

#### 解析配置

| 配置项 | 说明 | 来源 |
|--------|------|------|
| `ocr_model` | VLM 模型用于图片 OCR | 知识库配置 / 系统默认 |
| `asr_model` | ASR 模型用于语音转写 (默认: `paraformer-zh`) | 知识库配置 / 系统默认 |
| `richText` | 是否启用富文本解析 (保留标题层级) | 知识库配置 |

#### 分块策略 (Hierarchical Chunking)

| 层级 | 大小 | 说明 |
|------|------|------|
| Parent Chunk | ~1000 字符 | 语义完整段落，用于 LLM 上下文 |
| Child Chunk | ~200 字符 | 精确向量检索单元 |

- 仅 Child Chunk 被嵌入存储到 Milvus
- Parent Chunk 存储在 MySQL，检索时通过 `parent_id` 关联
- 检索返回 Parent Chunk 内容作为上下文

### 2.3 向量检索

#### 检索类型

| 类型 | 说明 | 前置条件 |
|------|------|----------|
| 纯向量检索 | ANN 向量最近邻 | Milvus 可用 |
| 纯关键词检索 | BM25 风格分词匹配 | 无 |
| 混合检索 | alpha × 向量分 + (1-alpha) × 关键词分 | Milvus 可用 |
| 多模态检索 | 图片 → VLM 描述 → 混合检索 | VLM 模型配置 |

#### 检索参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `topK` | 3 | 返回结果数量 |
| `alpha` | 0.7 | 向量权重 (0=纯关键词, 1=纯向量) |
| `threshold` | 0.0 | 相似度阈值过滤 |
| `rerankModel` | null | Rerank 模型 (启用需配置) |
| `embeddingModel` | null | 指定 Embedding 模型 |

#### 降级行为

| 场景 | 降级行为 |
|------|----------|
| Milvus 不可用 | 自动切换到纯关键词检索 |
| VLM 未配置 | 图片返回 `[图片解析失败: VLM 模型未配置]` |
| SiliconFlow API Key 未配置 | 音频返回 `[音频转写失败: 未配置 SiliconFlow API Key]` |
| FFmpeg 不可用 | 视频返回 `[视频解析需要 FFmpeg 支持]` |

### 2.4 知识图谱

| 能力 | 说明 | 状态 |
|------|------|------|
| 图谱 CRUD | 创建、查询、更新、删除图谱 | ✅ 可用 |
| 触发构建 | 手动触发图谱构建 | ✅ 可用 |
| 实体查询 | 分页获取图谱实体列表 | ✅ 可用 |
| 关系查询 | 分页获取图谱关系列表 | ✅ 可用 |
| 实体搜索 | 按关键词搜索图谱实体 | ✅ 可用 |

### 2.5 外部同步

| 来源 | 能力 | 前置条件 |
|------|------|----------|
| RAGFlow | 测试连接、列知识库、同步文档、检索、上传 | RAGFlow 服务可用 |
| Dify | 同步知识库列表 | Dify API Key 配置 |
| Notion | 测试连接、列数据库、同步页面 | Notion API Key |
| Web | 测试 URL、同步网页内容 | Jina Reader 可用 |
| Database | 连接测试、获取 Schema、同步表数据 | JDBC 配置 |

## 三、前置条件

### 3.1 必需依赖

| 服务 | 环境变量 | 说明 |
|------|----------|------|
| MySQL | `DB_HOST`, `DB_PORT`, `DB_NAME`, `DB_USERNAME`, `DB_PASSWORD` | 知识库元数据存储 |
| Milvus | `MILVUS_HOST`, `MILVUS_PORT`, `MILVUS_TOKEN` | 向量检索存储 |
| SiliconFlow | `SILICONFLOW_API_KEY` | LLM、Embedding、ASR、VLM 调用 |

### 3.2 可选依赖

| 服务 | 环境变量 | 影响的 capability |
|------|----------|-------------------|
| Redis | `REDIS_HOST`, `REDIS_PORT` | 缓存加速 (当前未强制依赖) |
| FFmpeg | 系统安装 | 视频音频提取 |
| RAGFlow | `RAGFLOW_DEFAULT_ENDPOINT`, `RAGFLOW_DEFAULT_API_KEY` | RAGFlow 外部同步 |
| Jina Reader | `JINA_READER_API_KEY` | 网页内容抓取 |

### 3.3 Embedding 模型要求

| 模型 | 维度 | 用途 |
|------|------|------|
| BAAI/bge-m3 | 1024 | 默认向量模型 |

**重要**: Embedding 模型维度必须与 Milvus Collection 维度匹配，否则检索返回空结果。维度不匹配时 `/knowledge/diagnose/milvus` 接口会返回 `dimensionMismatch: true` 警告。

## 四、已知限制

### 4.1 解析限制

| 限制 | 说明 | 规避方案 |
|------|------|----------|
| 扫描版 PDF | 无文字层，纯图片 PDF 无法解析 | 先 OCR 处理或使用文字版 PDF |
| DOC 格式 | 老版本 DOC 支持较弱 | 转换为 DOCX 格式 |
| 大文件 | 当前无显式的文件大小限制，但会全量加载到内存 | 拆分大文件或控制单文件体积 |
| 音频/视频时长 | 无硬性限制，受 API 超时和费用影响 | 控制单文件时长在 30 分钟内 |

### 4.2 检索限制

| 限制 | 说明 |
|------|------|
| Parent-Child 检索 | 仅对新数据生效 (向量记录包含 `parent_id` 元数据) |
| 向量维度不匹配 | 导致检索返回空结果 |
| Rerank 模型 | 需要额外配置 `rerankModel` 参数 |

### 4.3 外部同步限制

| 限制 | 说明 |
|------|------|
| RAGFlow 同步 | 仅支持同步文档，不支持实时增量同步 |
| Notion | 同步范围受 Notion API 权限限制 |
| 数据库同步 | 需要明确的表结构和主键 |

### 4.4 知识图谱限制

| 限制 | 说明 |
|------|------|
| 图谱构建 | 需手动触发，不支持定时自动构建 |
| 图谱与知识库关联 | 图谱数据与知识库文档无自动同步机制 |

## 五、能力边界速查表

### 5.1 什么可以做

- ✅ 上传 PDF/Word/图片/音频/视频文件到知识库
- ✅ 上传后自动解析、自动向量化
- ✅ 使用文字或图片进行语义检索
- ✅ 调整向量/关键词混合权重 (alpha 参数)
- ✅ 对检索结果进行 Rerank 重排
- ✅ 从 RAGFlow / Dify / Notion / Web / 数据库同步数据
- ✅ 手动构建知识图谱并查询实体/关系
- ✅ 为智能体绑定多个知识库 (DOCUMENT / STRUCTURED / API 类型)

### 5.2 什么不能做

- ❌ 上传时无显式文件大小限制 (依赖服务器内存)
- ❌ 对已解析文档的增量更新 (需重新上传)
- ❌ 知识图谱自动从知识库内容构建 (需手动触发)
- ❌ 跨知识库的联合检索 (需在应用层聚合)
- ❌ 自动识别并跳过加密/损坏文件

## 六、错误处理对照表

| 错误信息 | 原因 | 处理方式 |
|----------|------|----------|
| `[图片解析失败: VLM 模型未配置]` | 未配置 VLM OCR 模型 | 在知识库设置或系统设置中配置 `ocr_model` |
| `[音频转写失败: 未配置 SiliconFlow API Key]` | SiliconFlow API Key 未配置 | 配置 `SILICONFLOW_API_KEY` |
| `[视频解析需要 FFmpeg 支持]` | FFmpeg 未安装或不在 PATH | 安装 FFmpeg |
| `[PDF解析失败]` | PDF 文件损坏或加密 | 重新获取文件或移除密码保护 |
| `dimensionMismatch: true` | Embedding 维度与 Collection 不匹配 | 重建 Collection 或更换 Embedding 模型 |
